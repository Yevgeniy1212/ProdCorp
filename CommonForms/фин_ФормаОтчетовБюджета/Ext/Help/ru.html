Общие настройки отчетов

Для большинства отчетов используются стандартные настройки, применяемые в полном или частичном объеме. Настройки отчета изменяются в его форме на закладках, расположенных в правой ее части.

На закладке «Данные» указываются следующие настройки:

·         Граница актуальности – дата, по состоянию на которую будут взяты данные. Позволяет вывести данные, имеющиеся только на указанную дату, без учета более поздних корректировок плана или более поздних исполнений по факту;

·         Организация – доступно для заполнения, если ведется бюджетирование по организациям. Если поле заполнено, то выводятся данные только по указанной организации;

·         Разделитель данных – в этом поле можно выбрать один из аналитических разрезов данных отчета (разрезы планирования бюджетов или другие разрезы, доступные для структуры отчета). В этом случае результат вывода отчета будет представлен в виде серии отдельных таблиц, каждая из которых будет сформирована по отдельному значению указанного разреза;

·         Отбор – в данном поле можно настраивать отбор данных отчета по аналитическим разрезам. Для каждой строки отбора указывается признак использования, поле отбора, вид сравнения и значение сравнения.

Закладка «Структура» предназначена для описания колонок и ресурсов отчета.

В табличном поле «Группировки» настраиваются разрезы, в которых будут отображаться данные отчета, т.е. колонки и строки выводимой таблицы отчета. Можно создавать вложенные группировки (т.е. подчиненные колонки или строки), если создано несколько группировок для отображения в строках или колонках. При этом их вложенность зависит от порядка расположения в данном поле. В строке данного поля заполняются следующие колонки:

·         Группировка – аналитический разрез, доступный для отчета, который следует выводить в поле отчета. Выбирается из списка;

·         Вид группировки – способ отображения выбранной группировки в структуре отчета:

o    Колонка – значения группировки будут выведены в колонках отчета;

o    Колонка с иерархией – аналогично предыдущему виду группировки, но в случае, если группировка представлена иерархическим справочником, значения группировки будут выведены в колонках с учетом их иерархии в справочнике и с указанием групп справочника;

o    Группировка строк – значения группировки будут выведены в строках отчета;

o    Группировка строк с иерархией - аналогично предыдущему виду группировки, но в случае, если группировка представлена иерархическим справочником, значения группировки будут выведены в строках с учетом их иерархии в справочнике и с указанием групп справочника;

o    Группировка: только иерархия – значения группировки будут выведены в строках отчета, при этом если группировка представлена иерархическим справочником, будут выведены не отдельные элементы справочника, а их группы, с учетом их иерархии в справочнике;

·         Расположение – имеет значение в случае наличия больше одной группировки строк в отчете, определяет положение значений выбранного разреза относительно группировки более высокого уровня:

o    Вместе – значения текущей группировки будут располагаться под значениями группировки верхнего уровня, в той же колонке;

o    Отдельно - значения текущей группировки будут располагаться под значениями группировки верхнего уровня, но в соседней колонке. При этом для каждой строки со значениями текущего разреза слева будет продублировано значение группировки более высокого уровня, к которому относятся текущие строки;

o    Отдельно и только в итогах – аналогично предыдущему варианту расположения, только в строках со значениями текущего разреза не дублируются значения верхнего уровня группировки;

o    Применять оформление – признак, определяющий, будут ли поля выбранной группировки оформляться стандартно, либо к ним будут применяться шрифт и цвет фона заливки, указываемые в колонках «Шрифт» и «Цвет» соответственно;

·         Составное поле – данный признак устанавливается, если выводимое значение группировки должно состоять из более чем одного значения, например, вложенные поля. Если данный признак установлен, тогда в колонке «Состав» настраивается список выводимых в группировке полей и порядок их следования. Под представлением «ЗначениеПоля» подразумевается непосредственное значение выбранной группировки. Также могут выбраны вложенные реквизиты поля (если системе удается определить тип группировки) или другие разрезы отчета и их вложенные поля. Также необходимо заполнить значение в колонке «Расположение реквизитов», которое будет определять, каким образом будут выводиться отдельные составные поля данного разреза:

o    Вместе с владельцем – все составные поля будут выведены в одной ячейке, через запятую;

o    В специальной позиции – для составных полей будут автоматически созданы отдельные колонки, размещаемые в конце отчета, либо над колонками вложенных группировок. Расположение определяется системой автоматически;

o    Отдельно – каждое составное поле будет выведено в отдельной колонке, все они будут расположены рядом друг с другом;

o    Вместе – основное значение группировки будет выведено в отдельной колонке, а все прочие составные поля будут объединены в одну общую колонку, выводимую рядом;

В поле «Показатели» перечисляются те числовые показатели, которые могут быть выведены в качестве значений ячеек отчета.

На закладке «Дополнительно» настраиваются некоторые индивидуальные параметры отчета, касающиеся его оформления, а также следующие параметры, общие для большинства отчетов:

·         Язык отчета – выбирается язык, на котором должен быть сформирован отчет, если настроен перевод для выбранного языка;

·         Применять разрядность – для отчетов, формируемых по фактическим данным бюджетов, включает применение разрядности планирования к выводимым данным;

·         Группы статей под статьями/ Итоги по счетам снизу – для отчетов по бюджетам позволяет располагать группы статей / счетов не над подчиненными строками, а под ними;

·         Выводить коды строк бюджета – для отчетов по бюджетам позволяет отображать коды строк бюджета в отчете;

·         Упорядочивание статей – для отчетов по бюджетам определяет способ сортировки статей в отчете – по наименованию, коду строки бюджета или коду элемента справочника статей бюджетов;

·         Ширина колонка показателя – определяет ширину колонок, в которых будут выводиться показатели отчета;

·         Ширина колонки реквизита - определяет ширину колонок, в которых будут выводиться группировки отчета;

·         Ширина колонки статьи бюджета - определяет ширину колонок, в которых будут выводиться статьи бюджета или счета плана счетов бюджетирования;

·         Показатели в одной колонке – определяет расположение показателей отчета – либо в соседних колонках, либо друг под другом.

Также доступны следующие команды:

·         Печать – открывает окно печати поля отчета;

·         Отправить – доступна, если используется почтовый клиент, и позволяет отправить сформированный отчет по электронной почте;

·         Сохранить значения – позволяет сохранить созданные настройки формирования отчета для последующего использования. Если в создаваемой настройке указать признак «Использовать при открытии», то эта настройка будет применяться в дальнейшем к этому отчету при его открытии и формирование отчета будет также выполняться в этот момент. Если установить для настройки признак «Сохранять автоматически», тогда при закрытии отчета такая настройка будет автоматически заменяться на имеющиеся в форме отчета настройки. Настройки хранятся в справочнике «Сохраненные настройки (упр.)». При сохранении настройки в ней записывается пользователь, создавший ее. При необходимости можно добавить в настройку список пользователей, которые могут ей пользоваться. В противном случае настройка будет доступна только ее автору;

·         Восстановить значения – позволяет загрузить сохраненные настройки отчета.
