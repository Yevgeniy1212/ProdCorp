Настройка параметров отчетности налогового мониторинга

Конфигурация   "1С-Рейтинг: Налоговый мониторинг".

 

Форма предназначена для настройки параметров конфигурации "1С-Рейтинг: Налоговый мониторинг".

 

Поле "Номер версии конфигурации" не доступно для редактирования и предназначено для отображения текущего номера версии конфигурации "1С-Рейтинг: Налоговый мониторинг".

Поле "Основная конфигурация" предназначено для выбора конфигурации, в которую встроено дополнение "1С-Рейтинг: Налоговый мониторинг". Например, если дополнение встроено в конфигурацию "Бухгалтерия для Казахстана" или в конфигурацию, представляющую из себя отраслевое решение, построенное на базе конфигурации "Бухгалтерия для Казахстана", то необходимо выбрать значение из списка "Бухгалтерия для Казахстана". Если дополнение встроено в конфигурацию "Управление производственным предприятием для Казахстана", то соответственно выбирается значение "Управление производственным предприятием для Казахстана". Исключение составляет конфигурация "1С-Рейтинг: Нефтебаза", для которой предусмотрен отдельный пункт, поскольку в дополнении предусмотрен ряд механизмов специально адаптированных для конфигурации "1С-Рейтинг: Нефтебаза", например, значение пин-код в отчетах будет заполняться из первичных документов, а не из регистра Объекты мониторинга. Поле обязательно для заполнения.

 

Флажок "Заполнять коды ТНВЭД из справочника "Номенклатура" - если флажок установлен, то при записи элемента справочника "Номенклатура" будет выполнятся автоматическое добавление записей в регистр сведенй "Объекты мониторинга" с заполнением реквизита "Код ТНВЭД" по значению из справочника.

Табличное поле "Правила заполнения форм" предназначено для установки правил заполнения форм которые следует применять только начиная с определенной даты (с детализацией равной месяцу):

  • Использовать данные из дополнительных счетов фактур - флажок, устанавливающий необходимость получать сведения о количестве, стоимости товара, сумме НДС и сумме акциза из дополнительного счета-фактуры, введенного на основании первичного документа, отражающего операции приобретения\реализации в бухгалтерском учете. Данное правило применяется при заполнении следующих отчетов:
    • МОП: Форма 1.1 Книга реализации по дополнительному счету-фактуре (НП);
    • МОП: Форма 2.1 Книга покупок по дополнительному счету-фактуре (НП);
    • МОП: Форма 1.1.2 Книга реализации по дополнительному счету-фактуре (ХС);
    • МОП: Форма 1.2.2 Книга покупок по дополнительному счету-фактуре (ХС);

Для того, чтобы изменения настройки вступили в силу, требуется перепроведение документа Движения мониторинга.

Пример:

В документе "Возврат ТМЗ от покупателя" в табличной части "ТМЗ" отражена операция возврата товара от покупателя в количестве 1шт на сумму 1000 тг. На основании данного документа сформирован дополнительный счет-фактура выданный, где в табличной части "ТМЗ" указывается количество возвращенного товара, равное -1 на сумму -1000тг, т.е. значения указаны с отрицательным знаком, при этом сумма НДС и сумма Акциза также будут отрицательными. Если в табличном поле "Правила заполнения форм" формы настройки параметров установить флажок "Использовать данные из дополнительных счетов фактур", то в отчетах "МОП: Форма 1.1 Книга реализации по дополнительному счету-фактуре (НП)" и "МОП: Форма 1.1.2 Книга реализации по дополнительному счету-фактуре (ХС)" значения граф "Количество", "Размер корректировки", "Акциз" и "НДС" будут указаны с отрицательным знаком (как указано в дополнительном счете-фактуре), в противном случае - с положительным.

Сохранение измененных параметров выполняется после нажатия кнопки "ОК".
