Установка даты запрета изменения данных

В конфигурации поддерживается возможность установки ограничения изменения данных ранее определенной даты (далее "дата запрета").

Контроль ограничения изменения данных выполняется для:

  • всех документов при проведении и отмене проведения;
  • наборов записей периодических регистров сведений, с независимым от регистратора режимом записи;
  • наборов записей регистров накопления, при этом контроль выполняется во всех случаях изменения записей регистров, включая программное.

При этом для документов проверяемым значением является дата этого документа, для наборов записей - период каждой из записей данного набора.

При установке даты запрета следует учитывать, что под действие данного ограничения попадают данные, период действия которых ограничен концом предыдущего дате запрета дня. То есть при установке значения 31.12.2004 изменение данных будет возможно только начиная с 01.01.2005 00:00:00 включительно.

Дата запрета устанавливается в разрезе пользователей и групп пользователей. В случае если пользователь одновременно включен в несколько групп, в качестве ограничивающей даты берется минимальная из дат, установленных для этих групп, то есть та дата, которая наименее ограничивает пользователя.

Ограничение по дате запрета не распространяется на пользователей, которым доступна роль "Полные права", если не стоит параметр учета "Применять дату запрета для пользователей с "Полными правами"".

Различаются понятия "основной даты запрета" и "даты ограничения изменения регламентной информации". Соответственно, в область действия основной даты попадают все данные, вторая дата ограничивает изменение регламентной информации, раздельно для каждой из организаций. При этом, если дата запрета для регламентной информации не задана, при контроле изменения этой информации будет применяться основная дата.  Если одновременно не задано значение и для основной даты и для регламентной - контроль изменения данных не будет задействован.

"Основная дата запрета" устанавливается на закладке "Основная дата" формы. Отображение заданных значений в разрезе пользователей и групп пользователей организовано в виде дерева, что позволяет задать значение как для всех пользователей(групп пользователей) одновременно (строка дерева "Общая дата"), так и для каждого пользователя(группы пользователей) в отдельности.

Установка ограничения на изменение регламентной информации выполняется на закладке "Ограничение регламентной информации". В случае необходимости установки даты запрета отдельно для конкретной организации, в дереве можно указать дату запрета как в целом для всей организации, так и для каждого пользователя(группы пользователей) в отдельности. Для указания даты для конкретного пользователя необходимо в древе выбрать нужную организацию и в меню добавить выбрать добавление пользователя либо группы пользователей. В случае, когда необходимость указания отдельной даты запрета для организации пропала, в соответствующей строке дату нужно удалить.

Примеры использования.

Пример 1. Указана основная дата запрета (31.01.2004), ограничение регламентной информации не задействовано.

В этом случае любое изменение как документов, так и отдельных записей регистров с датой ранее 01.02.2004 не доступно.

Пример 2. Указана основная дата запрета (31.12.2003), для организации "Торговый дом "Комплексный" отдельно указана дата 31.05.2004.

Изменение данных других организаций доступно только после 01.01.2004. Изменение данных по организации "Торговый дом "Комплексный" только после 01.06.2004. Например, документ "Поступление товаров и услуг" датой 01.03.2004 нельзя будет провести по организации "Торговый дом "Комплексный", но возможно по любой другой.
