Отчет "Пользовательский отчет"

Для формирования произвольного отчета используется отчет "Пользовательский отчет", в котором в поле «Вид отчета» выбирается созданный шаблон и настраивается использование его полей. В списке «Параметры» заполняются те параметры, которые предусмотрены в шаблоне отчета для ввода пользователем.

При настройке шаблона указывается его наименование, а также один из главных параметров – «Вид источника», который определяет порядок настройки шаблона и может принимать одно из значений:

·         Объект информационной базы – данные будут взяты из указанного объекта информационной базы. Самый простой вид отчета;

·         Произвольный запрос – данные будут взяты из созданного пользователем запроса любой сложности;

·         Объединение шаблонов – данные будут объединены из нескольких других шаблонов отчетов.

Далее рассматриваются все указанные виды отчетов.

Источник – объект информационной базы

Для данного вида шаблона в поле «Источник» выбирается таблица информационной базы, из которой необходимо производить выборку данных.

После выбора источника дальнейшая настройка производится на основном поле формы путем переключения между разделами.

Для переключения между ними используется навигационная панель «Настройка источника данных отчета».

Пункт «Настройка отбора данных» используется для того, чтобы настроить фильтр по полям объекта информационной базы. В поле «Настройка отбора данных» при необходимости добавляются строки отбора, где в колонке «Поле отбора» указывается поле объекта или его вложенное поле, в колонке «Вид сравнения» указывается способ его сопоставления со значением отбора, а в колонке «Параметр» указывается текстовое представление параметра, значение которого будет передаваться в качестве фильтра. Передача самих значений настроенных параметров настраивается в разделе «Ввод параметров отбора» панели навигации.

В этом разделе для тех имен параметров, которые настроены в разделе «Настройка данных отбора», указывается способ определения их значения. Также здесь, в зависимости от выбранного объекта информационной базы, будут доступны дополнительные параметры, например, параметры определения периода выборки для виртуальных таблиц регистров системы.

В колонке «Тип параметра» определяется тип вводимых значений. В колонке «Список» определяется необходимость ввода нескольких значений (списка). В колонке «Заполнять в отчете» определяется необходимость заполнения значения данного параметра пользователем в момент формирования отчета. В противном случае следует указать фиксированное значение в колонке «Значение», либо в окне списка для списочных параметров. Эти значения будут постоянными для параметра и не будут изменяться пользователем.

Поля отчета, которые будут выводиться пользователю, определяются в разделе «Настройка выводимых полей отчета» панели навигации.

В списке «Поля отчета» вводится перечень выводимых полей. В колонке «Поле» создается поле, описание которого хранится в справочнике «Поля пользовательских отчетов».

В элементе данного справочника в поле «Наименование» указывается его представление. В поле «Роль поля» определяется, будет ли это поле измерением (группировкой отчета) или ресурсом (его показателем). Если это ресурс, то следует выбрать итоговую функцию, при помощи которой будут определяться значения по группировкам отчета.

В поле «Способ отображения» выбирается одно из значений:

·         По пользовательскому представлению – данное поле отчета будет называться так, как написано в поле «Наименование»;

·         По соответствию измерениям бюджетирования – данное поле отчета будет называться аналогично разрезу планирования бюджетов, выбранному в поле «Измерение».

Поле «Сортировка» определяет необходимость и порядок сортировки данных отчета по значениям поля.

Когда колонка «Поле» заполнена, необходимо заполнить следующие колонки:

·         Тип поля – определяет источник данных поля:

o    Поле источника – непосредственное данные из того поля объекта информационной базы, которое будет выбрано в колонке «Имя поля»;

o    Фиксированное значение – будет выводиться фиксированное значение из колонки «Значение»;

o    Выражение – будет выводиться результат расчета по формуле, описанной в поле «Выражение» в формате языка запросов. В данной формуле можно обращаться по именам к числовым полям источника отчета.

Источник – произвольный запрос

В случае, если источником является произвольный запрос, тогда в разделе «Создание запроса к информационной базе» панели навигации производится создание запроса с возможностью использования конструктора запросов.

Значения параметров, заданных в запросе, устанавливаются в разделе «Ввод параметров запроса».

Их настройка осуществляется так же, как и в случае использования в качестве источника объекта информационной базы. Таким же образом производится настройка выводимых полей в разделе «Настройка выводимых полей отчета».

Источник – объединение шаблонов

В случае, если источником является объединение шаблонов, в разделе «Выбор интегрируемых шаблонов» панели навигации выбираются другие шаблоны, представляющие собой обращение к произвольным запросам, либо объектам информационной базы, которые будут объединены в рамках данного шаблона. Это позволяет декомпозировать составление сложных отчетов на несколько более простых с последующим объединением.

В разделе «Ввод параметров шаблонов» приводится список тех параметров вложенных шаблонов, для которых установлен признак «Заполнять в отчете». Здесь порядок их заполнения можно переопределить, передав фиксированное значение.

Затем в разделе «Настройка выводимых полей отчета» создаются его поля, аналогично предыдущим типам источников.

В колонке «Имя поля» для тех полей, которые выбираются из полей источника, следует ввести программное имя для его использования при компиляции запроса. Имя служебное и не влияет на логику выполнения отчета.

Затем следует вернуться в раздел «Соответствия полей шаблонов» и настроить соответствия между полями текущего отчета, выбираемыми из вложенных, и полями каждого вложенного шаблона.

После этого отчет готов к исполнению.
