Справочник "Маршруты согласования и утверждения документов"

Предназначен для ведения списка маршрутов, с указанием настроек маршрута. Каждый маршрут может использоваться только для одной позиции классификатора документов. Для маршрута указывается способ его использования и перечень этапов, для каждого из которых производятся дополнительные настройки.

В форме маршрута изображена карта маршрута, обновляемая автоматически при изменении состава этапов, либо по кнопке «Обновить». При работе в режиме «тонкого клиента» форма немного отличается, при этом она содержит меньше функциональных возможностей в части работы с контекстным меню поля карты.

На закладке «Карта маршрута» настраиваются этапы маршрута и их характеристики при помощи кнопок меню и контекстного меню поля карты. В карту может быть добавлено два типа элементов:

·         Этап – это этап маршрута, определяющий действия пользователей по согласованию;

·         Переход – это разветвление маршрута в зависимости от условий, применяемых для адресации документа между ветками перехода. Проверка условий и адресация документа при прохождении разветвления выполняется автоматически.

При добавлении этапов и переходов, у каждого из них автоматически или вручную указывается их уровень – то есть порядок удаленности от начала маршрута. Переход от этапов меньшего уровня осуществляется только к этапам большего уровня. При необходимости уровень этапа или перехода можно изменить.

Для добавления этапа необходимо выбрать одну из команд правой командной панели либо контекстного меню:

·         Добавить следующий – этап будет добавлен на следующий уровень после выделенного. Для него сразу будет установлено соединение с выделенным на текущий момент этапом. Если добавление производится при выделенном переходе, а не этапе, то система предложит выбрать одно из условий перехода, определяющее, какую ветку перехода будет продолжать добавленный этап;

·         Добавить произвольный – этап будет добавлен на следующий уровень после выделенного, но соединение с другими этапами не будет установлено;

При этом будет открыта карточка этапа, описывающая его характеристики (см. «Карточка этапа»).

Для добавления перехода следует использовать команду «Добавить переход». Будет добавлен переход на следующий уровень и открыта карточка перехода (см. «Карточка перехода»).

Настройка соединений с другими этапами производится либо из карточек этапов и переходов, либо при помощи кнопки «Установить соединение с этапом-приемником» после выбора этапа-источника связи. Затем следует щелкнуть по этапу-приемнику. Или выбрать его и нажать кнопку «Принять соединение». После этого связь будет установлена. Если в качестве источника выбран переход, то система предложит выбрать одну из его веток. При добавлении соединений следует контролировать, чтобы не было нескольких соединений, исходящих от одного этапа или одной и той же ветки условного перехода.

Команда «Карточка этапа» открывает карточку выделенного этапа/перехода.

Команда «Удалить этап» удаляет выбранный этап/переход с разрывом маршрута в этом месте.

Команда «Заменить этап» позволяет заменить выбранный этап на другой, с сохранением соединений и прочих свойств.

После того, как по маршруту начинают двигаться документы, его редактирование становится доступным только пользователям с полными правами после нажатия кнопки «Редактировать».

На закладке «Применение маршрута» настраиваются общие характеристики маршрута:

·         Использование маршрута

o    По умолчанию – маршрут будет применяться всегда, независимо от параметров конкретного документа;

o    Использовать при условии – маршрут будет применяться в том случае, если выбранное условие из справочника «Условия выполнения операций по документам» выполняется применительно к конкретному документу, для которого определяется маршрут;

·         Порядок возврата на пересмотр – определяет порядок программного выбора этапов, на которые документ будет возвращаться в случае возврата на пересмотр с одного из следующих этапов

o    Настраиваемый порядок возврата на пересмотр – для каждого этапа вручную будет настроен этап, на который его необходимо вернуть в случае возврата на пересмотр;

o    Последовательный порядок возврата на пересмотр – документ по умолчанию будет возвращаться на этап маршрута, предшествующий текущему.

Для механизма возвратов предусмотрен предопределенный этап маршрутов «Возврат инициатору маршрута», который можно выбрать в качестве этапа возврата. Также этот этап используется при возврате с первого этапа маршрута и означает возврат автору документа для доработки.

·         Финализация маршрута – определяет действия, выполняемые программно после завершения последнего этапа маршрута:

o    Финальное состояние – состояние из справочника Состояния документов, присваиваемое при финализации;

o    Финальное корректировка реквизитов документа – признак, определяющий необходимость автоматически изменить значения реквизитов, указанных в списке «Корректируемые при финализации реквизиты документа»;

·         Защита прошедших маршрут документов от корректировок – настраивает возможность корректировки документа после его отклонения или финализации маршрута пользователями, не имеющими прав согласования на финальном этапе:

o    Запретить корректировку после прохождения – документ будет доступен для редактирования только пользователям, имеющим права согласования на последнем пройденном этапе;

o    Разрешить корректировку после прохождения – документ будет доступен всем пользователям, имеющим права доступа к документу, независимо от данных согласования;

·         Актуальность маршрута – управляет записями для данного маршрута в регистре Актуальные маршруты согласования документов:

o    Используется в данный момент – маршрут добавляется в список актуальных маршрутов;

o    Не используется в данный момент – маршрут удаляется из списка актуальных маршрутов;

В списке этапов маршрута описываются все использованные этапы. Не следует изменять эту табличную часть без особой необходимости.

На закладке «Дополнительно» настраиваются при необходимости следующие характеристики маршрута:

·         Режим возврата на пересмотр – порядок адресации документа при его возврате:

o    Автор документа – при возврате документ всегда отправляется автору;

o    Предыдущий согласующий – при возврате документ будет отправлен на тот этап и тому пользователю, где он был перед текущим исполнителем;

o    По выбору исполнителя – возвращающий сможет самостоятельно определить, куда возвращается документ;

·         Правило продолжения маршрута при пересмотре – если документ согласуется на том этапе, на который он был возвращен, данное значение определяет, куда он отправится на дальнейшее согласование:

o    Продолжить маршрут с этапа пересмотра – документ пойдет дальше, следуя порядку маршрута после текущего этапа;

o    Отправить на этап, с которого был возвращен документ – документ будет отправлен на тот этап, с которого его вернули;

o    Отправить лично исполнителю, вернувшему документ – документ будет отправлен на тот этап, с которого его вернули, именно тому пользователю, который его вернул;

·         Также можно перенастроить прочие реквизиты, определяющие пропорции размещения элементов на карте маршрута.
