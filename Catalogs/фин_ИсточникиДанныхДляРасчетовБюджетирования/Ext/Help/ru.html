Справочник «Источники данных для расчетов бюджетирования»

В системе предусмотрен механизм для настройки пользователем правил выборки данных из базы конфигурации. Такие настройки составляют справочник «Источники данных для расчетов бюджетирования» (меню «Источники» - «Источники данных для расчетов бюджетирования»).

Переключатель «Источник данных» служит для указания  типа источника:

·         Объект информационной базы – выбранный регистр сведений (или таблица среза по нему), регистр накопления или бухгалтерии (и их таблицы оборотов и остатков);

·         Произвольный запрос – текст  запроса, сформированный пользователем.

Настройка источников данных может проводиться тремя способами:

·         в пользовательском режиме: выборка настраивается параметрически, аналогично настройке данных универсальных отчетов;

·         в пользовательском режиме, однако, выборка настраивается не параметрически, а написанием непосредственно кода запроса к данным;

·         в режиме конфигуратора: запрос составляется для предопределенных элементов справочника в определенном формате (шаблоне предоставления выходной информации).

После окончания настройки источника в любом режиме можно проверить результат выборки данных (по кнопке Проверить).

Такие шаблоны выборки данных используются в подсистеме бюджетирования для:

·         задания источника для цены в описании статьи оборотов;

·         задания источника для получения фактических данных по статье оборота;

·         задания базы в профилях изменения плановых бюджетных данных по измерениям;

·         расчетов по моделям бюджетирования;

·         ввода и коррекции остатков на бюджетном плане счетов.

Источник данных выбирается в реквизите «Источник».

Формирование запроса в Конфигураторе

Если запрос к базе данных создается в Конфигураторе, то такой запрос следует поместить в код модуля формы элемента справочника «Источники данных для расчета бюджетирования». В пользовательском режиме в справочнике в качестве источника следует выбрать раздел «Отборы, определенные пользователем».

Формирование запроса в пользовательском режиме

Если текст запроса нужно написать не в Конфигураторе, а в пользовательском режиме, то в форме справочника следует нажать кнопку Произвольный запрос. В форме появится закладка «Запрос» для ввода данных запроса и кнопка вызова конструктора запроса.

В текстовом поле на закладке «Запрос» с помощью конструктора или вручную формируется текст запроса; по кнопке с галочкой выполняется заполнение на основании текста запроса полей источника, при этом в поле «Показатели» переносятся все выбранные поля, имеющие числовой тип.

Параметры запроса делятся на статические и динамические:

·         статические — параметры, которые не изменяются в зависимости от периода выполнения запроса по источнику; указываются на закладке «Параметры» аналогично консоли отчетов;

·         динамические — параметры:

o     имеющие тип «Дата», которые подставляются в момент выполнения запроса по источнику: указываются в таблице «Границы отбора данных» на закладке «Данные» аналогично указанию дат при работе в режиме автоматической настройки выборки данных источника; дополнительно указывается имя, как оно задано в запросе; указание динамических параметров производится вручную.

o    Параметры, указанные на закладке «Параметры» не фиксированными значениями, а ссылками на элемент перечисления «Передаваемые параметры источников данных при расчете моделей», позволяющий вычислять значение динамически. Может быть использовано только для источников, используемых в регламентированных моделях.

После этого, настройка источника (связь с измерениями, отборы) аналогична работе в режиме автоматической настройки выборки данных источника.

 

Важно!!! Существует ограничение: в тексте произвольного запроса нельзя использовать конструкцию «Итоги»: для суммирования значений необходимо использовать группировки.

Параметрическая настройка запроса

Для параметрической настройки запроса к базе источник выбирается также в поле «Источник», в окне выборки можно указать источник из следующих групп:

·       разделы управленческого учета (регистры накопления);

·       справочная информация (регистры сведений);

·       остатки и обороты по планам счетов (планам счетов бухгалтерского, налогового учета, плану счетов бюджетирования).

Основные принципы создания шаблонов и заполнения справочника будут рассмотрены в следующих разделах.

Определение периода выборки

Для выбранного объекта настраивается период выборки данных на закладке «Отбор» в табличной части «Границы отбора данных».

Для регистров накопления, регистров бухгалтерии, предопределенных отборов настраивается начальная и конечная дата периода, для регистров сведений одна дата — дата получения данных.

В шаблоне выбора периода для начальной и конечной даты проектируемого запроса можно указать один из следующих способов их определения, в поле «Вид границы»:

·       «Фиксированная дата» - начальная и / или конечная дата являются фиксированными датами, значения которых устанавливаются пользователем, в этом случае независимо от того, на какой период ведется бюджетирование, при обращении к шаблону запроса будет выбираться один и тот же период, заданный в шаблоне;

·       «Начало периода» - начальная дата совпадает с началом периода бюджетирования, в рамках которого производится обращение к данному источнику данных (шаблону запроса);

·       «Конец периода» - конечная дата совпадает с концом периода бюджетирования, в рамках которого производится обращение к данному источнику данных (шаблону запроса);

Дополнительно можно указать периодичность и шаг сдвига границ отбора данных по отношению к моменту выборки. Сдвиг может быть осуществлен как вперед, так и назад. Например, при планировании стоимости продаж, запланированных на август, можно оценить плановый объем по средней цене предыдущего месяца. В этом случае периодом бюджетирования будет август, а шаблон запроса (источника данных) для определения средней цены фактически произведенных продаж должен быть задан со сдвигом на один месяц раньше относительно периода бюджетирования.

Также, при получении данных по источнику, если для границы отбора указан вид «Начало периода», то переданная дата начала отбора будет приведена к началу дня, а если же указан вид «Конец периода», то переданная дата окончания отбора будет приведена к концу дня.

Определение числовых показателей в шаблоне

Перечень возможных для использования показателей зависит от того, какого типа осуществляется выборка из базы:

·       если выборка производится из регистров сведений, накопления, регистров бухгалтерии, то параметрами являются соответствующие виды ресурсов этих регистров;

·       если выборка осуществляется для предопределенного набора, то параметры задаются при проектировании самого отбора.

Показатели выбираются на закладке «Данные» в группе реквизитов «Показатели».

В дальнейшем при обращении к шаблону запроса пользователь может выбрать из списка доступных полей запроса ресурс, который необходим для конкретной ситуации использования шаблона. В самом шаблоне можно указать, какие из возможных полей будут доступны в дальнейшем пользователю при обращении к шаблону. Например, при определении шаблона получении отпускной цены номенклатуры, мы используем регистр сведений «Цены номенклатуры». Для этого регистра определены два ресурса: цена и процент скидки/наценки. Предполагая, что в дальнейшем при обращении к такому шаблону нас не будет интересовать ресурс «Процент скидки/наценки», мы можем его исключить из списка доступных полей в самом шаблоне.

Связь аналитических разрезов запроса с измерениями бюджетирования

Для использования данных из шаблонов запроса нужно, по возможности, связать доступные измерения запроса с аналитикой по бюджетированию, то есть с:

·       ЦФО;

·       проектом;

·       сценарием;

·       контрагентом;

·       номенклатурой;

·       валютой;

·       организацией.

 

Важно!!! Связь может устанавливаться не для всех видов аналитики, а лишь для тех, которые будут иметь значение при дальнейшем использовании шаблона.

Связь с измерениями бюджетирования указывается на закладке «Данные» в табличной части «Связь с измерениями бюджетирования».

При указании связи данных запроса и аналитики по бюджетированию можно использовать достаточно гибкий механизм настройки. В самом простейшем случае аналитике по бюджетированию будут соответствовать данные измерений регистров, по которым делаются запросы. В более сложном случае для связи могут использоваться реквизиты объектов (измерений), участвующих в запросе. Например, если запрос осуществляется по регистру «Денежные средства», то для связи по аналитике бюджетирования «Валюта» можно использовать реквизит «Валюта» измерения «БанковскийСчетКасса» этого регистра.

Фактически при настройке связи с аналитическими разрезами бюджетирования можно использовать любые объекты, которые можно получить по цепочке ссылок из данных, вошедших в шаблон запроса.

Отбор данных в запросе

При формировании шаблона запроса можно установить правила отбора (фильтры) записей из общего результата запроса. Настройка отборов может проводиться для любых объектов, к которым можно получить доступ по цепочке ссылок из данных, вошедших в запрос.
