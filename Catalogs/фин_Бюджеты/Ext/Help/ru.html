Справочник «Бюджеты»

Список бюджетов определяется в справочнике «Бюджеты» (меню «Бюджеты» - «Бюджеты»). Каждый отдельный элемент этого справочника определяет отдельный бюджет. Их количество и структура не ограничены.

При создании/редактировании/просмотре бюджета открывается специальная форма, состоящая из нескольких закладок, переключаемых кнопками на левой панели формы.

Настройка основных параметров бюджета

На закладке «Основное» заполняются основные характеристики бюджета – его наименование (как он отображается в документах), заголовок (как он отражается в отчетах). Можно составить подробный комментарий по назначению бюджета.

В группе «Использование» определяется, необходимо ли создавать отдельные экземпляры бюджетов на каждое подразделение/проект, либо один экземпляр бюджета будет охватывать все подразделения/проекты.

Если в этой группе выбран вариант «Без принадлежности к разрезу», тогда в одном экземпляре бюджета будут вводиться данные по всем аналитическим разрезам.

Если выбран вариант «Бюджет проекта», тогда на каждый проект будет создаваться отдельный экземпляр бюджета. При этом в поле рядом можно указать основной проект, который всегда будет подставляться системой автоматически при создании экземпляра данного бюджета.

Если выбран вариант «Бюджет подразделения», тогда на каждое подразделение будет создаваться отдельный экземпляр бюджета. При этом в поле рядом можно указать основное подразделение, который всегда будет подставляться системой автоматически при создании экземпляра данного бюджета. При выборе значения данного поля открывается окно настройки подразделения.

 

Если установлен признак «Для списка подразделений», тогда система позволит указать список подразделений, которым будет разрешено оформлять экземпляр данного бюджета. На каждое подразделение будет создаваться отдельный экземпляр.

Если заполнено только поле «Подразделение по умолчанию», то оно будет просто использоваться для автоподстановки в новых экземплярах. Каждый экземпляр будет создаваться на отдельное подразделение. При этом можно будет указать отличное от выбранного.

Если заполнено поле «Подразделение по умолчанию» и указан признак «Включая подчиненные подразделения», тогда на указанное подразделений будет создаваться один экземпляр бюджета, в котором будут заполняться данные в разрезе его подчиненных подразделений.

В группе «Оформление» страницы «Основное» указывается необходимость использования кодов строк бюджета, а также поле, используемое для сортировки его статей.

В некоторых случаях процесс составления бюджета должен быть организован по заявочной системе, когда подразделения подают заявки, и они объединяются в бюджет. При этом иногда бывает необходимым ограничить для каждого подразделения список тех статей бюджета, которые они могут заполнять. Для этого используется команда «Порядок формирования заявок по подразделениям» верхней командной панели, открывающая регистр сведений «Порядок формирования заявок подразделений по бюджетам».

В данном регистре сведений указывается перечень статей бюджетов и подразделения, которые имеют право заполнять их. В дальнейшем, когда будет оформляться заявка на формирование бюджета от подразделения, для которого настроены такие ограничения, в заявке будут видны только выбранные статьи. Если ограничений по подразделению не предусмотрено, то в заявке от него буду видны все статьи бюджета.

Аналитические разрезы статей бюджета

Перечень аналитических разрезов планирования, которые можно будет использовать для его статей, определяется на закладке «Разрезы бюджета».

 

Здесь могут быть выбраны любые доступные в системе разрезы планирования бюджетов. Порядок добавления бюджетов также играет роль, т.к. определяет вложенность разрезов при вводе экземпляров бюджетов.

Если бюджет используется как бюджет проекта или подразделения, то соответствующий разрез добавляется автоматически и не может быть отключен.

Дополнительные настройки бюджета

Некоторые дополнительные настройки самого бюджета, а также настройки, общие для всех его статей, задаются на закладке «Дополнительно».

 

В поле «Режимы расчета итоговой строки бюджета» можно настроить расчет общего итога по бюджету. Может быть выбран один из вариантов:

·         Не рассчитывать – специальных итоговых строк рассчитываться не будет;

·         По итоговой функции – в отдельном поле можно будет указать итоговую функцию (сумма, среднее и т.д.), применяемую для расчета итога. В расчете не участвуют статьи, для которых установлен признак «Исключить из итогов»;

·         По формуле – в отдельном поле можно будет указать формулу, применяемую для расчета итогов.

В группе «Параметры статей бюджета по-умолчанию» определяются показатели и разрезы учета, устанавливаемые всем вновь создаваемым статьям бюджета. После создания значения по-умолчанию можно будет изменять.

Установленный признак «Версионировать по-умолчанию» означает, что если у бюджета уже созданы его версии, то для каждая новая строка автоматически будет включена в состав новой версии, а не в состав постоянных статей бюджета (см. пункт «Версионирование бюджетов»).

Структура статей бюджета

Для бюджета может быть настроена произвольная структура статей. Его статьи могут быть выстроены в иерархическую структуру любой вложенности. При этом планирование будет вестись по статьям непосредственно, а группы статей служат лишь для агрегации данных по входящим в их состав статьям.

Структура статей бюджета выстраивается на закладке «Статьи бюджета».

Каждой конечной статье бюджета (не идет речь о группах статей) устанавливается в соответствие финансовый показатель, в разрезе которого и будут храниться данные в системе.

Заполнение финансового показателя происходит несколькими способами:

·         Путем заполнения колонки «Финансовый показатель» в строке с созданной статьей бюджета. При этом, если наименование статьи не было заполнено, оно будет взято из финансового показателя. Также признаки учета статьи будут взяты из настроек финансового показателя;

·         После записи бюджета автоматически создаются финансовые показатели для тех статей, для которых колонка «Финансовый показатель» не заполнена;

·         Путем перетаскивания элементов справочника «Финансовые показатели» в нужную группу статей. В этом случае будет добавлена новая статья бюджета, заполненная по перетаскиваемому показателю.

Создание статей происходит двумя способами:

·         При помощи команд «Добавить статью» и «Добавить группу статей»;

·         Либо, как уже было сказано, перетаскиванием элементов или групп элементов из справочника «Финансовые показатели».

Для каждой статьи указывается код строки (если учет кодов строк включен для бюджета), признаки учета (количественный/суммовой) и разрезы, а также способ расчета цены, если ведется учет и по сумме и по количеству. Данные признаки хранятся в финансовом показателе, поэтому настройки финансовых показателей обновляются в случае изменения данных признаков. Если финансовый показатель участвует в разных бюджетах, то отключение разреза в статье данного бюджета приведет к его отключению и в статьях других бюджетов.

Если ведется использование остаточных и рассчитываемых финансовых показателей, то для статьи также можно указать вид ее показателя и формулу, если показатель рассчитываемый. Колонка «Итоговая функция» определяет функцию расчета итогов на основании данных аналитических разрезов (если это отдельная статья), либо подчиненных статей (если это группа статей).

Колонка «Исключить из итогов» определяет, следует ли не учитывать статью или группу статей в итогах по вышестоящей группе.

Колонка «Уровень элиминации» используется для определения того, при какой детализации данная статья имеет смысл в сводном бюджете. Если значение не заполнено, то статья не подвергается элиминации. Возможные значения:

·         Всегда – данная статья не будет участвовать в отчетах по бюджету и его исполнению;

·         На уровне организации – данная статья будет отображаться в отчетах только при наличии разреза по организации, при его формировании по всем организациям она будет считаться внутренним оборотом и не будет отображаться;

·         На уровне подразделения – данная статья будет отображаться в отчетах только при наличии разреза по подразделениям, при его формировании по всем подразделениям она будет считаться внутренним оборотом и не будет отображаться.

Хранение самого списка статей бюджета осуществляется в справочнике «Статьи бюджета», заполняемом автоматически.

Версионирование бюджетов

При эксплуатации бюджетов на протяжении длительного срока может понадобиться внесение изменений с сохранением структуры бюджетов, используемой ранее. Например, с определенного периода в структуре бюджета появляются или удаляются, переименовываются статьи. Для того, чтобы сохранить на каждый период индивидуальную структуру одного и того же бюджета используется версионирование. Для этого создается элемент в справочнике «Версии бюджетов», в котором указвается представление версии и дата ее вступления в силу. Каждый бюджет может иметь собственный перечень версий, либо он может быть сквозной. Также можно версионировать только выбранные бюджеты. Бюджет начинает версионироваться с того момента, как для одной или более его статей указывается версия (колонка «Версия» перечня статей бюджета). Те статьи, у которых версия не указана, будут присутствовать во всех его версиях. Те же статьи, у которых она настроена, будут считаться версионируемыми и относиться только к конкретной версии. Если бюджет версионируется, то при его открытии на закладке «Статьи бюджета» будет отражаться его последняя версия. Если указать другую версию, то будут отображены статьи, входящие в состав той версии.

При создании/редактировании экземпляра бюджета применяемая версия будет определяться на основании того, какая версия была в силе на начало планового периода, охватываемого бюджетом.
