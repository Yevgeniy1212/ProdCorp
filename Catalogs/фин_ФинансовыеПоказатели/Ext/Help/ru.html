Справочник «Статьи оборотов по бюджетам»

В регистрах системы плановые и фактические данные хранятся не в разрезе статей бюджета, а в разрезе финансовых показателей, закрепленных за статьями бюджетов. Статьи бюджетов, в свою очередь, связываются с данными финансовыми показателями. Таким образом, финансовый показатель представляет отражение какого-либо факта финансово-хозяйственной деятельности предприятия, а статья бюджета, за которой он закреплен, определяет лишь вывод данного показателя в каком-либо бюджете. Один и тот же финансовый показатель может фигурировать в нескольких бюджетах. Такой подход уменьшает объем вводимой и хранимой информации, а также обеспечивает идентичность строк бюджетов в тех случаях, когда несколько строк различных бюджетов оперируют одним и тем же фактом финансово-хозяйственной деятельности предприятия.

Описанный подход удобен для регистрации фактических данных в интерактивном режиме, если для документа необходимо указывать вручную финансовый показатель. В таком случае нет необходимости при заполнении документа задумываться о бюджетах, в которых должен отразиться документ. Вместо этого указывается финансовый показатель, единый для всех бюджетов, а отражение в нужных бюджетах система берет на себя.

Перечень финансовых показателей хранится в данном справочнике.

Рассмотрим основные настройки этого справочника.

Настройки учета по финансовому показателю

На закладке «Основное» можно настроить разрезы и признаки учета. Данные настройки будут применяться ко всем статьям бюджетов, соотнесенным с данным финансовым показателем.

В группе «Признаки учета» определяется в каком выражении необходимо планировать и контролировать данные по статье:

·         Учет по количеству – необходимость ведения по статье учета в количественном выражении;

·         Учет по сумме – необходимость ведения учета по статье в денежном выражении;

·         Основная валюта – валюта, подставляемая по умолчанию при планировании по статье.

В группе «Используемые разрезы учета» настраиваются разрезы планирования и учета фактических данных по данному финансовому показателю (см. пункт «Аналитические разрезы бюджетирования»).

Если в параметрах учета включена опция ведения учета по плановой номенклатуре, а в финансовом показателе добавлен разрез «Номенклатура», то под списком разрезов учета становится доступным признак «Учет по плановой номенклатуре», который позволяет планировать данные по финансовому показателю в разрезе справочника «Плановая номенклатура».

В том случае, если по финансовому показателю уже имеются плановые данные или прочие настройки, то использованные ранее разрезы учета становятся недоступными для удаления/изменения, и можно только добавлять новые разрезы.

Расчет цен по финансовому показателю

В связи с тем, что при работе со статьями бюджета, ведомыми и в суммовом и в количественном выражении, часто требуется ввод цены по позиции, реализован специальный механизм расчета цен. К таким статьям могут относиться, например, реализация или закуп товаров. Механизм расчета цен позволяет настроить для финансового показателя, закрепленного за какой-либо статьей бюджета, источник расчета цен. Затем, при вводе строк по данной статье, например, при вводе плана по отдельным номенклатурным позициям, планируемым к реализации или закупу, система автоматически подставит цену из данного источника, настроенную для выбранной номенклатурной позиции. Пользователю останется ввести количество, а сумма по статье будет рассчитана автоматически на основании данных о количестве и цене.

Настройка расчета цен доступна только для тех финансовых показателей, у которых установлен как учет по сумме, так и учет по количеству. В этом случае становится доступной для заполнения группа реквизитов «Расчет цен». Источник, который будет использовать система для определения цен, определяется значением поля «Способ ценообразования», которое может принимать одно из следующих значений:

·         Не рассчитывать – в этом случае автоматический расчет цены не производится;

·         По плановым ценам номенклатуры – цена будет определяться на основании плановых цен. В этом случае понадобится заполнить поле «Используемый тип цен», в котором указать элемент справочника «Типы цен», для которого были введены плановые цены;

·         По бухгалтерским ценам номенклатуры – цена будет определяться на основании регистра «Цены номенклатуры», имеющегося в типовых решениях. Если такого регистра в основной конфигурации нет, этот способ недоступен. При использовании этого способа понадобится заполнить поле «Используемый тип цен», в котором указать элемент справочника «Типы цен», для которого были введены цены;

·         Фиксированная (в валюте упр. учета) – цена будет единая по всем плановым операциям по данному показателю, независимо от того, какая номенклатура указана, и ведется ли учет в разрезе номенклатуры по показателю вообще. В этом случае цена и дата, на которую она актуальна, могут быть указаны в этой же форме. При изменении цены с какой-либо даты, можно будет указать новую цену и дату. Актуальные цены и их история хранятся в регистре сведений «Фиксированные цены по финансовым показателям». Этот способ можно использовать, в частности, в показателях без разреза по номенклатуре, например «Электроэнергия, КВт.» и т.п.

Виды финансовых показателей

По-умолчанию, создаваемые финансовые показатели считаются оборотными, т.е. показателями, по которым просто отражается величина плана/факта за выбранный период. И на каждый период по показателю свои обороты плановых/фактических данных.

В том случае, если в параметрах учета устанавливается признак «Вести учет остаточных/расчетных финансовых показателей», то в форме финансового показателя становится доступным для заполнения поле «Вид показателя».

В системе доступны следующие виды показателей:

·         Оборотный – обычный финансовый показатель, по которому пользователь может ввести плановую величину (сформировать плановые обороты), и позднее будет зарегистрирован факт (сформированы фактические обороты);

·         Остаточный – финансовый показатель, значение которого не вводится пользователем, а рассчитывается как остаток между увеличивающими его другими финансовыми показателями и между уменьшающими;

·         Рассчитываемый – финансовый показатель, значение которого не вводится пользователем, а рассчитывается на основании других финансовых показателей по настроенной формуле.

Настройка остаточных финансовых показателей

Если для финансового показателя установлен вид «Остаточный», тогда становится доступной для заполнения закладка «Расчет остатков по показателю».

На этой закладке настраиваются правила расчета остатков – какие финансовые показатели формируют остатки по данному показателю. На остатки могут влиять только оборотные финансовые показатели.

Для каждой строки настройки указывается дата ввода строки в действие, сам финансовый показатель, а также его вид влияния на остаток:

·         Приход – обороты по данному показателю будут увеличивать остатки;

·         Расход – обороты по данному показателю будут уменьшать остатки;

·         Не отражается  – обороты по данному показателю не будут влиять на остатки. Может пригодиться, если ранее показатель отражался на остатках, а с какого-то периода необходимо прекратить его влияние с сохранением истории изменений, на случай пересчета прошлых периодов.

При добавлении новой строки настроек (или нескольких строк) для нее устанавливаются настройки из последней добавленной строки (период действия и вид влияния на остаток) для удобства работы с групповым добавлением строк.

Групповое добавление строк становится возможным при использовании команды «Подбор», если выбрать в ней не отдельный финансовый показатель, а группу справочника. Тогда в настройки будут добавлены все оборотные показатели, входящие в группу и в подчиненные ей группы. Им будут установлены настройки, идентичные последней добавленной перед ними строке.

Расчет остатков по такому финансовому показателю осуществляется в пределах периода. То есть на размер остатка в определенном периоде планирования влияют только те обороты по финансовым показателям, которые также относятся к данному периоду. Таким образом, остаток является остатком в пределах периода. Для организации перетекающих остатков, то есть остатков, учитывающих не только обороты текущего периода, но и остатки предыдущего, следует использовать рассчитываемые финансовые показатели.

Настройка рассчитываемых финансовых показателей

Если для финансового показателя установлен вид «Рассчитываемый», тогда ему автоматически устанавливается признак учета по сумме и отключается учет по количеству, а также становится доступной для заполнения поле «Формула». В данном поле выбирается элемент справочника «Формулы расчета финансовых показателей».

Дополнительные команды работы с финансовыми показателями

В верхней командной панели формы присутствуют дополнительные команды:

1.       Заметки – позволяет работать с заметками и комментариями по элементу справочника;

2.       Проводки по счетам бюджета – доступна, если ведется учет по плану счетов бюджетирования (задается параметрами учета), и позволяет настроить порядок отражения данных по финансовому показателю на управленческом плане счетов;

3.       Регулярный контроль показателя – позволяет настроить правила регулярной автоматической установки лимитов по показателю;

4.       Отложенный сбор факта – отображается, если в параметрах учета не отключен учет фактических данных в отложенном режиме. Позволяет настроить правила отражения фактических данных в режиме off-line.
