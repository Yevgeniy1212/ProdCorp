Версия конфигурации

На этой закладке справочно показывается номер текущей версии типовой конфигурации.

Номер версии изменяется автоматичеки при обновлении конфигурации, вручную он не редактируется. Процесс обновления конфигурации можно запустить, нажав на ссылку "Информация об обновлениях".

Заголовок главного окна

При запуске программы в заголовке её главного окна отображается название конфигурации, наименование организации, заданной в качестве основной и наименование пользователя. Если одновременно происходит работа с несколькими информационными базами, удобнее заменить этот стандартный заголовок на такой, который поможет быстро определить, какая именно из информационных баз сейчас открыта. Чтобы это сделать, нужно ввести нужный текст в поле "Заголовок" на этой закладке. 

Обмен данными

На закладке задаются параметры для автоматического обмена данными при работе с распределенными информационными базами.

Префикс узла для распределенной информационной базы - задается, если предполагается обмен данными между распределенными информационными базами (РИБ). Префиксация позволяет избежать дублирование кодов элементов в узлах РИБ.

Префиксация осуществляется только если в списке узлов плана обмена "Полный" создан хотя бы один узел. В этом случае префикс добавляется к кодам при создании новых элементов справочников, документов и элементов планов видов характеристик. Установка или изменение префикса не приводит к перенумерации кодов уже существующих элементов.

Пользователь - определяет пользователя, в клиентском сеансе которого должны выполняться фоновые и регламентные задания в файл-серверном режиме работы информационной базы. Рекомендуется запускать отдельный клиентский сеанс от имени указанного пользователя. В частности, это позволяет производить автоматический обмен по расписанию. При этом пользователь должен обладать достаточными правами для выполнения произвольных операций по чтению и записи данных информационной базы.

Для клиент-серверного режима работы этот параметр задавать не требуется.

Интервал опроса регламентных заданий в файловом режиме также необходимо задавать только для файл-серверного режима работы информационной базы.

Права доступа

Если требуется предоставить разным пользователям права к разным элементам общих справочников (Организации, Физические лица и т.д.) необходимо установить флажок "Использовать ограничения прав доступа на уровне записей". После установки флажка необходимо применить данную настройку (по кнопке "ОК") и перезапустить программу. Затем необходимо снова открыть настройку программы на этой закладке и нажать на кнопку "Настроить права доступа". В открывшемся списке групп пользователей нужно настроить права доступа, используя кнопку "Права" верхней командной панели списка.

Запрет изменения данных

С помощью кнопки "Установить дату запрета" можно установить дату, ранее которой (включая саму дату запрета изменения данных) изменять любые данные в информационной базе будет запрещено. Дату запрета можно установить одну для всей информационной базы, либо установить разные даты запрета для разных организаций или групп пользователей.

Пользователи, имеющие полные права, по умолчанию могут изменять данные ранее даты запрета. Если требуется запретить изменять данные и этим пользователям тоже, необходимо установить флажок "Распространить дату запрета на пользователей с полными правами".

Каталог дополнительной информации

В поле "Каталог" указывается каталог на диске, в котором хранятся внешние обработки и документы, полученные средствами встроенного веб-обозревателя.
