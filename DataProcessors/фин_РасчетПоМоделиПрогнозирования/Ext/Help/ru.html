Обработка «Расчет по модели прогнозирования»

Для расчета прогнозных данных предназначена обработка «Расчет по модели прогнозирования». В данном мастере расчет состоит из следующих шагов:

1.       Настройка модели – определение ее основных параметров: вида модели, прогнозируемого показателя, правил сбора исходных данных;

2.       Обучение модели – получение исходных данных модели (автоматически или вручную) и выполнение программных расчетов параметров модели в зависимости от выбранного вида;

3.       Выполнение прогноза – расчет прогнозных данных на основании результатов обучения модели;

4.       Сохранение настроек модели (при необходимости дальнейшего использования).

В поле «Вид модели прогнозирования» указывается одно из следующих значений, определяющих способ расчета прогноза:

·         Линейная модель множественной регрессии – используется в том случае, если рассматривается связь между результирующим финансовым показателем и значениями других (влияющих) финансовых показателей. Расчет производится при помощи регрессионного анализа. Исходными данными для модели служат значения влияющих и результирующего финансовых показателей за наблюдаемый период. Регрессионная модель описывает некоторую закономерность. Линейная регрессионная модель подразумевает зависимость результирующего показателя от влияющих, которую можно выразить в линейном виде. При этом для построения модели используются измеряемые данные, а не знание свойств исследуемой закономерности;

·         Нелинейная модель множественной регрессии – в отличие от предыдущего вида модели подразумевает нелинейную зависимость между значениями результирующего и влияющих показателей;

·         Регрессионная модель временного ряда (методом наименьших квадратов) – прогноз значений финансового показателя, который можно осуществить на основе измеренных значений этого показателя на некотором отрезке времени. При проведении прогноза используются уравнения регрессии, параметры которых находятся на основе метода наименьших квадратов. В этом случае рассматривается регрессионная зависимость показателя от периодов наблюдения. Для точности прогноза следует использовать данные измерений за равные интервалы времени;

·         Модель прогнозирования временного ряда на основе экспоненциального сглаживания - прогноз значений финансового показателя, который можно осуществить на основе измеренных значений этого показателя на некотором отрезке времени. При проведении прогноза используется метод экспоненциального сглаживания.

Далее рассмотрены варианты работы с мастером в зависимости от выбранного вида модели прогнозирования. Выбор вида модели осуществляется пользователем на основании выбранного им способа и знания о закономерностях динамики показателя.

Линейная модель множественной регрессии

Если необходимо производить расчет прогнозных данных по регрессионной модели, коэффициенты (оценки параметров) которой уже известны (рассчитаны ранее и т.д.), то устанавливается признак «Фиксированные оценки параметров». Если же оценки параметров неизвестны и их необходимо рассчитывать на основании исходных значений влияющих показателей, тогда флажок не устанавливается и необходимо указать, каким образом будут получены исходные данные для анализа. Для этого заполняются следующие поля:

·         Способ сбора наблюдаемых данных:

o    Фиксированные значения – исходные значения результирующего и влияющих показателей за требуемое количество периодов будут введены пользователем вручную (например, если данных за прошлые периоды еще нет в системе);

o    Значения заданного периода - исходные значения результирующего и влияющих показателей будут получены системой автоматически из данных бюджетирования. При этом период обучения модели будет фиксированным и не будет зависеть от того, на какой период выполняется прогноз;

o    Значения периода, предшествующего прогнозируемому - исходные значения результирующего и влияющих показателей будут получены системой автоматически из данных бюджетирования. При этом период обучения модели будет определяться автоматически в зависимости от того, на какой период выполняется прогноз. Т.е. будут браться данные, имеющиеся за нужное количество предшествующих прогнозному периодов;

·         Количество наблюдаемых периодов – какое количество периодов необходимо для обучения модели;

·         Организация – организация, данные по которой следует использовать (если ведется бюджетирование по организациям);

·         Сценарий наблюдаемых данных – сценарий, данные которого будут использованы для обучения модели:

o    Выбранный сценарий – наблюдаемые данные будут браться по фиксированному сценарию, независимо от того, для какого сценария выполняется прогноз;

o    Прогнозируемый сценарий – наблюдаемые данные будут браться по тому же сценарию, для которого выполняется прогноз;

·         Наблюдаемый период – заполняется в том случае, если в качестве значения поля «Способ сбора наблюдаемых данных» выбрано «Значения заданного периода». Определяет период сбора данных для обучения модели.

В группе «Параметры прогнозируемых данных» определяется результирующий показатель модели и интервал прогноза:

·         прогнозируемый показатель – финансовый показатель, являющийся результирующим для модели;

·         прогнозируемый ресурс показателя – какой ресурс (количество, сумма) будет прогнозироваться в рамках модели;

·         сценарий прогнозных данных – сценарий, данные которого по выбранному показателю необходимо прогнозировать;

·         Количество прогнозируемых периодов;

·         Прогнозируемый период – первый период, на который составляется прогноз.

В поле «Комментарий» можно ввести текстовые пояснения к модели, если ее планируется сохранить для дальнейшего использования.

После того, как настроены основные параметры модели, при помощи кнопки «Далее» осуществляется переход на следующий шаг.

На этом шаге в списке «Исходные показатели модели» указываются наблюдаемые финансовые или расчетные показатели, зависимость от которых анализируется. В колонке «Лаг» указывается запаздывание, экономический показатель, характеризующий временной интервал между двумя взаимосвязанными экономическими явлениями, одно из которых является причиной, а второе - следствием. Например, существует лаг между началом производственного выпуска товаров и их массовой продажей, выделением капиталовложений на строительство и вводом в действие строительных объектов. Используемый в экономико-математических моделях распределенный лаг учитывает наличие разных промежутков времени между разными частями явления-следствия и явления-причины. Например, принимается во внимание, что после выпуска партии товаров она поступает в продажу частями с разными интервалами времени (лагами).

В списке «Отбор по разрезам планирования» можно настроить фильтр по значениям разрезов планирования.

Начиная с этого шага, становится доступным сохранение настроек модели в справочнике «Модели прогнозирования» для последующего расчета на основании сохраненных ранее настроек. Выполняется сохранение при помощи кнопки «Сохранить модель» нижней командной панели.

При переходе на следующий шаг производится получение данных для обучения модели (если не выбран способ сбора наблюдаемых значений вручную).

В поле мастера выводятся полученные значения (либо, если данные вводятся вручную, только таблица для их ввода). Полученные автоматически данные также можно исправить. Помимо этого необходимо в группе «Прогнозируемые значения исходных показателей» ввести предполагаемые значения исходных показателей на прогнозируемые периоды. На основании этих значений и оценок регрессионной модели будут получены прогнозные величины.

Данную страницу можно распечатать при помощи кнопки «Печать». Также на данном шаге можно выполнить корреляционный анализ между результирующим и исходными показателями для оценки степени влияния и корректности выбора показателей в качестве исходных. Анализ производится при нажатии на кнопку «Корреляционный анализ».

В данном анализе выводятся парные коэффициенты корреляции между исходными и результирующим показателями, а также между каждым из исходных показателей. В колонках напротив каждого показателя указываются его парные коэффициенты корреляции: в колонке r xy - коэффициент корреляции с результирующим показателем, r xxN – коэффициент корреляции с исходным показателем номер N. Для возвращения в мастер следует использовать кнопку «Назад».

При переходе на следующий (последний) шаг осуществляется обучение модели и расчет прогнозных данных. Результат прогнозирования можно распечатать. На этом шаге можно воспользоваться командой «Статистические свойства». В выведенном отчете будут указаны рассчитанные оценки параметров модели а также будет выполнена оценка погрешности модели при ее применении к данным наблюдаемого периода.

Нелинейная модель множественной регрессии

Расчет модели производится путем линеаризации модели, т.е. путем подбора таких преобразований для входящих показателей, которые позволят представить искомую зависимость в виде линейного соотношения между преобразованными переменными.

В связи с этим, отличие от настроек линейной регрессионной модели заключается в том, что на первом шаге в группе полей «Параметры прогнозируемых данных» становятся доступны для заполнения поля:

·         Преобразование входящих значений – формула линеаризации для значений результирующего показателя за наблюдаемый период. Можно не заполнять, если не требуется;

·         Преобразование выходных значений – формула преобразования результирующих прогнозных значений с учетом того, что изначально результат прогноза рассчитывается по линеаризованной модели, и, возможно, потребуется преобразование выходных значений. Можно не заполнять, если не требуется.

На втором шаге в списке «Исходные (наблюдаемые) показатели модели» также появляется колонка «Формула преобразования», которая будет использоваться для линеаризации значений определенного исходного показателя. Можно не заполнять, если не требуется.

В указанных выше полях для ввода формул при их изменении открывается окно ввода формулы, в котором само значение показателя, которое подлежит линеаризации, обозначено именем переменной «Значение». В формуле можно использовать те же операции и конструкции, что и во встроенном языке 1С:Предприятие.

Регрессионная модель временного ряда (методом наименьших квадратов)

Для анализа временного ряда необходимо указать, каким образом будут получены исходные данные для анализа. Для этого заполняются следующие поля:

·         Способ сбора наблюдаемых данных:

o    Фиксированные значения – исходные значения результирующего и влияющих показателей за требуемое количество периодов будут введены пользователем вручную (например, если данных за прошлые периоды еще нет в системе);

o    Значения заданного периода - исходные значения результирующего и влияющих показателей будут получены системой автоматически из данных бюджетирования. При этом период обучения модели будет фиксированным и не будет зависеть от того, на какой период выполняется прогноз;

o    Значения периода, предшествующего прогнозируемому - исходные значения результирующего и влияющих показателей будут получены системой автоматически из данных бюджетирования. При этом период обучения модели будет определяться автоматически в зависимости от того, на какой период выполняется прогноз. Т.е. будут браться данные, имеющиеся за нужное количество предшествующих прогнозному периодов;

·         Количество наблюдаемых периодов – какое количество периодов необходимо для обучения модели;

·         Организация – организация, данные по которой следует использовать (если ведется бюджетирование по организациям);

·         Сценарий наблюдаемых данных – сценарий, данные которого будут использованы для обучения модели:

o    Выбранный сценарий – наблюдаемые данные будут браться по фиксированному сценарию, независимо от того, для какого сценария выполняется прогноз;

o    Прогнозируемый сценарий – наблюдаемые данные будут браться по тому же сценарию, для которого выполняется прогноз;

·         Наблюдаемый период – заполняется в том случае, если в качестве значения поля «Способ сбора наблюдаемых данных» выбрано «Значения заданного периода». Определяет период сбора данных для обучения модели.

В группе «Параметры прогнозируемых данных» определяется результирующий показатель модели и интервал прогноза:

·         прогнозируемый показатель – финансовый показатель, являющийся результирующим для модели;

·         прогнозируемый ресурс показателя – какой ресурс (количество, сумма) будет прогнозироваться в рамках модели;

·         сценарий прогнозных данных – сценарий, данные которого по выбранному показателю необходимо прогнозировать;

·         Количество прогнозируемых периодов;

·         Прогнозируемый период – первый период, на который составляется прогноз.

В поле «Комментарий» можно ввести текстовые пояснения к модели, если ее планируется сохранить для дальнейшего использования.

После того, как настроены основные параметры модели, при помощи кнопки «Далее» осуществляется переход на следующий шаг.

В списке «Отбор по разрезам планирования» можно настроить фильтр по значениям разрезов планирования.

При переходе на следующий шаг производится получение данных для обучения модели (если не выбран способ сбора наблюдаемых значений вручную).
