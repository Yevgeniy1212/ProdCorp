Обработка "Формирование договоров по заказам поставщикам"

Регистрация договора по заказу(-ам) производится с помощью обработки "Формирование регистрации договоров по заказам поставщикам". Если зарегистрировать договор необходимо только по одному заказу, то можно воспользоваться вводом на основании.

При формировании договоров по заказам поставщикам с помощью мастера выбирается контрагент, и при нажатии на кнопку "Заполнить список заказов" в поле "Включаемые заказы" выводится список всех заказов поставщику, которые не были ранее включены в договоры.

В связи с тем, что регистрируемый договор должен содержать не плановые, а итоговые, бухгалтерские номенклатурные позиции (за исключением позиций по внеоборотным активам), при вводе регистрации договора на основании заказа поставщику, в котором присутствуют плановые номенклатурные позиции для товаров и услуг, происходит замена плановых позиций на указанные им в соответствии бухгалтерские позиции. В том случае, если для плановой номенклатуры не указано соответствие с бухгалтерской, то при таком вводе на основании открывается окно выбора регламентированной номенклатуры. В открывшемся окне приведен список плановых позиций, у которых не установлено соответствие с бухгалтерской номенклатурой.

В этой форме в поле "Регламентированная номенклатура" следует выбрать существующий элемент справочника "Номенклатура" или создать новый. Выбранный элемент будет установлен в соответствие плановой номенклатуре и помещен в создаваемый договор. Если не указать в этом окне регламентированную номенклатуру, тогда такая позиция заказа поставщику будет включена в спецификацию договора без указания номенклатуры.

Аналогичное окно будет открыто и в том случае, если для плановой номенклатуры есть соответствия, но их несколько, и программа не может однозначно определить бухгалтерскую номенклатуру, которую следует поставить в договор вместо указанной плановой.
