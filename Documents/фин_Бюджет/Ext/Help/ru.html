Документ «Бюджет»

 

Документ «Бюджет» является основным документом планирования бюджетов. В документе фиксируется планируемое движение средств предприятия по всем статьям определенного бюджета за период, продолжительность которого определяется периодичностью и горизонтом планирования сценария, выбранного для бюджета.

Доступны три вида операции:

·         Заявка - намерение включить в определенный бюджет необходимые показатели. Используется для планирования бюджета «снизу-вверх», когда, например, подразделения формируют ряд заявок на плановые данные, а затем менеджер на их основании формирует итоговый бюджет. Для подачи заявок предусмотрена специальная роль – «Право ввода заявок на формирование бюджетов». Для формирования бюджета на основании заявок используется обработка «Формирование бюджета». Заявка не формирует движений по регистрам;

·         Бюджет - непосредственно формируемый бюджет;

·         Корректировка – корректировка сформированного бюджета. При корректировке отображается полная картина бюджета на момент корректировки и пользователь может непосредственно редактировать представленные цифры чтобы получить тот результат, который ему необходимо;

·         Переоценка курсов - используется в том случае, если изменились курсы валют и требуется без изменения сумм операций пересчитать и откорректировать их величину в валюте управленческого учета или в валюте сценария. Рекомендуется использовать совместно с признаком "Задавать курсы валют в документе", в том числе и для первоначального бюджета;

Внимание!!! Бюджеты и корректировки бюджетов становятся недоступными для редактирования после проведения более поздних корректировок этого же бюджета.

В  документе в обязательном порядке указывается:

·         сценарий планирования;

·         период планирования;

·         бюджет, статьи которого планируются.

Также может быть указано подразделение или проект, бюджет которого формируется.

Признак "Единая валюта" позволяет указать валюту для всех операций только один раз, в шапке документа. Если он не установлен, тогда для каждой строки можно будет устанавливать собственную валюту. В таком случае итоги по группам статей и по строкам, имеющим подчиненные строки в разных валютах, рассчитываются в валюте сценария.

Признак "Задавать курсы валют в документе" позволяет заполнить и применять при расчетах и проведении фиксированные в документе курсы валют. Т.к. в первую очередь эти курсы используются для пересчета валютных сумм в валюту управленческого учета и сценария при проведении документа, данная возможность может быть полезна в случае частых колебаний курсов, если валюта, в которой планируются бюджеты, отличается от валюты управленческого учета. Это позволит проводить переоценку по новым курсам в последующем при помощи документов с видом операции "Переоценка курсов".

Указанные данные заполняются на странице "Основное".

При изменении основных параметров формируемого бюджета производится проверка на наличие бюджетов с аналогичными параметрами. Происходит перезаполнение бюджета входящими в его состав статьями. При этом статьи бюджета, ссылающиеся на финансовые показатели, которые используются в системе как зависимые или как рассчитываемые по моделям, не входят в структуру бюджета по умолчанию, и выводятся только при наличии оборотов (плановых показателей) по ним.

Важно!!! Невозможно сформировать несколько бюджетов с одинаковыми параметрами (бюджет, сценарий, подразделение) за указанный период и горизонт планирования. Таким образом, определенный бюджет за период, по определенному сценарию редактируется только в одном документе. Невозможно сформировать бюджет, не входящий в регламент.

Данные документа распределены на табличные части:

·         Обороты по статьям, введенные пользователем - хранит первичные обороты (плановые показатели) бюджета, вводимые в документе;

·         Зависимые обороты - хранит зависимые обороты (плановые показатели), рассчитанные от первичных оборотов бюджета. Заполняются автоматически каждый раз при записи документа, либо при ручном обновлении в процессе редактирования основных плановых показателей;

·         Результаты расчетов по моделям - хранит результаты расчета плановых показателей по моделям бюджетирования для статей бюджета, связанных со статьями оборотов, для которых заданы модели.

По умолчанию табличные части скрыты. Их видимостью можно управлять при помощи кнопки Показывать табличные части верхней командной панели.

Настройка внешнего вида поля редактирования данных бюджета

Управлять внешним видом поля редактирования данных бюджета можно при помощи настроек из группы "Оформление" на закладке "Дополнительно":

·         Вариант редактирования бюджета – позволяет выбрать вариант представления данных на закладке "Бюджет":

 

    • Иерархический - каждый последующий уровень аналитики отображается на следующем уровне (аналог поведения документа в редакции 2.6). На закладке в виде иерархического списка выводятся плановые показатели (обороты) по бюджету. Измерения бюджетирования представлены иерархически, каждое последующее измерение уточняет сумму предыдущего по дополнительным разрезам. Редактируются строки нижнего уровня иерархии. Показатели строк верхнего уровня суммируются на основании вложенных. ;
    • Таблицы с аналитикой в иерархии статей - иерархия строк дерева бюджета выстраивается только для статей. Дополнительная аналитика отображается в строках, в виде таблицы;
    • Табличный - иерархия бюджета не встраивается, все данные отображаются в виде таблицы;
    • Редактировать в табличных частях (без использования поля бюджета) - позволяет работать только с исходными табличными частями, без создания дерева данных бюджета;
    • Пользовательский вариант - позволяет редактировать данные бюджета в шаблоне, настроенном пользователем самостоятельно при помощи механизма настроек шаблонов отчетов, для отчета "Печатная форма документа "Бюджет";

 

·         Фиксация колонок бюджета – позволяет зафиксировать колонки с аналитикой таким образом, чтобы при прокручивании по горизонтали происходила прокрутка только для периодов, а не для аналитики;

·         Авто высота ячейки статьи, Авто высота ячеек аналитики – включает автоматическое изменение высоты ячеек таким образом, чтобы длинный текст умещался полностью;

Указанные настройки можно сохранить для последующего автоматического применения, выбрав в группе команд "Сохранение настройки редактирования" для всех ли пользователей и/или бюджетов необходимо применять сохренную настройку.

Заполнение бюджета

Основное редактирование данных производится на закладке «Бюджет».

Если по сценарию планирования предполагается редактирование данных нескольких периодов, тогда колонки дерева будут разбиты по периодам. Итоговая колонка также не будет редактироваться, а будет получаться суммированием показателей по горизонтали.

Порядок следования измерений (в т.ч. при наличии дополнительной аналитики) настраивается в форме бюджета.

Различные ячейки имеют определенный цвет:

·         Ячейки основных оборотов (плановых показателей), доступные для редактирования – светло-зеленый;

·         Ячейки основных оборотов (плановых показателей), недоступные для редактирования, либо получаемые суммированием по вертикали/горизонтали – желтый;

·         Строки зависимых оборотов (недоступны для редактирования) - включают как обороты (плановые показатели) сформированные в текущем документе, так и в других бюджетах, но относящиеся к текущему – серебристо-фиолетовый;

·         Строки расчета по модели (недоступны для редактирования) - включают расчеты по модели, произведенные в текущем документе – голубой;

·         Строки наследуемых оборотов (сформированные в бюджетах с более высоким приоритетам в рамках регламента) – розовый.

Важно!!! Если для финансового показателя, закрепленного за статьей бюджета, определен источник для формирования цены, то при выборе номенклатуры значение цены рассчитывается автоматически на основе данных источника. Далее, при установке планового количества значение суммового оборота может быть рассчитано системой, как Цена * Количество.

Для работы с данными бюджета используются следующие команды (работа с уровнями аналитики доступна только для иерархического варианта редактирования):

  • Добавить строку – добавляет строку того же уровня (того же разреза аналитики), что и выделенная строка. Для табличного варианта редактирования необходимо спозиционироваться на той колонке, значение которой нужно изменить в новой строке. Например, установив курсор мыши ячейке с проектом, можно добавить строку с другим проектом, но такими же значениями остальны разрезов;
  • Скопировать строку – работает по аналогии с командой Добавить строку, но при этом позволяет скопировать в нее числовые значения из той строки, на которой был спозиционирован курсор. Если у исходной строки имеются подчиненные строки, то они также будут скопированы;
  • Удалить строку – удаляет текущую строку. Если у исходной строки имеются подчиненные строки, то они также будут удалены;
  • Добавить уровень аналитики – добавляет разрез аналитики для текущей строки, если возможно. Разрезы аналитики последовательно детализируют строки бюджета. Таким образом, если в бюджете присутствует строка с указанной статьей бюджета, ее можно детализировать по любому аналитическому разрезу – уровню аналитики. Для строк определенного разреза аналитики доступна детализация только по разрезам, расположенным ниже его в настройке разрезов бюджета. При этом суммы, введенные по текущей строке, будут распределены по добавляемому измерению. При использовании данной команды открывается специальное окно.

В левой части окна в поле «Опции» приводится список разрезов аналитики, которыми можно расширить (уточнить) текущую строку. Если воспользоваться командой «По профилю», тогда список аналитических разрезов и их долей будет заполнен автоматически на основании элемента справочника «Профили изменения планов по измерениям». Можно произвести настройки одновременно для нескольких разрезов. При этом описанные настройки будут применены последовательно. При выборе разреза в поле «Опции», в правой части окна отображается страница с соответствующим разрезу заголовком. На этой странице указывается одно из 4 действий:

o    Установить значение – в добавленном разрезе будет присутствовать только одна строка. Значение аналитического разреза будет взято в поле списка значений в нижней части окна;

o    Тиражировать строку по списку позиций – на основании списка значений к заполнению разреза аналитики будет сформировано столько же строк данного разреза, сколько и значений в списке. Для данного способа, сумма, установленная в выделенной строке бюджета, будет продублирована для каждой добавленной строки (при этом умножена на долю, указанную в колонке «Доля» для каждого разреза);

o    Распределить строку по списку позиций – на основании списка значений к заполнению разреза аналитики будет сформировано столько же строк данного разреза, сколько и значений в списке. Для данного способа, сумма, установленная в выделенной строке бюджета, будет распределена между добавленными строками пропорционально долям, указанным в колонке «Доля» для каждого разреза;

o    Не изменять – по данному разрезу не будет произведено никаких действий.

Если выбраны действия по нескольким разрезам, то порядок действий будет выглядеть следующим образом: сначала формируются строки, распределенные по первому (более приоритетному) разрезу, затем полученные (в результате тиражирования/распределения или других действий) строки, каждая в отдельности, распределяются по следующему разрезу и т.д.

В поле «Опции» присутствует пункт «Отбор», позволяющий в случае необходимости применить распределение по измерениям не только для текущей строки, но и к другим строкам, путем изменения правил отбора, определяющих область выполнения запрошенных действий. Например, если необходимо применить распределение не к одной статье бюджета, а ко всем сразу. Следует применять внимательно.

Внимание!!! При использовании данной команды в том случае, если пользователь пытается добавить уровень аналитики, который уже заполнен для данной строки (например, данные по подразделениям уже введены по выбранной статье, а пользователь пытается применить к этой же статье данную команду для добавления разреза «Подразделение»), тогда существующие значения подразделений будут заменены на новые, а введенные для них уточняющие строки по вложенным разрезам аналитики будут перенесены в одну, с пустым значением подразделения.

 

·         Удалить уровень аналитики – удаляет разрез аналитики у текущей статьи бюджета. При этом показатели по разрезу суммируются и устанавливаются в качестве показателей верхнего уровня;

·         Распределить по периодам – открывает форму заполнения, которая при наличии нескольких плановых периодов позволяет заполнить данные в указанной строке, распределив величину на периоды пропорционально или по указанным пропорциям. Помимо распределения имеется возможность тиражировать (в т.ч. и с приращением) одну и ту же величину по периодам строки;

·         Подбор по значению колонки – позволяет заполнить уровень аналитики бюджета подбором. Например, если курсор установлен в колонке с номенклатурой, откроется форма подбора номенклатуры. пользователь сможет выбрать любое количество элементов справочника, которые будут добавлены на тот же уровень бюджета (с такими же значениями прочих разрезов);

·         Подбор по значению колонки (с копированием) – работает по аналогии с командой Подбор по значению колонки, но при этом позволяет скопировать в нее числовые значения из той строки, на которой был спозиционирован курсор. Если у исходной строки имеются подчиненные строки, то они также будут скопированы;

·         Очистить данные строки – очищает числовые показатели выбранной строки. Если у строки имеются подчиненные строки, они будут удалены;

·         Рассчитать бюджет – производит расчет по формулам, зависимостям, моделям и прочим настроенным расчетным связям. При необходимости раздельного расчета следует использовать команды из подменю Расчеты;

·        Установить состояние статей   – позволяет выбрать состояние и установить его для всех статей бюджета;

·        Проверить наличие лимитов по строке   – позволяет проверить наличие лимитов, установленных для операций планирования;

 

Команды работы с данными бюджета

Кнопка Расчет по моделям позволяет обновить данные расчетов по регламентным моделям, если существуют модели для расчета в контексте текущего документа  (то есть модели, задающие формирование плановых показателей по статьям редактируемого бюджета, для редактируемого сценария). После расчета по моделям открывается список выполненных расчетов. При изменении данных документа и при попытке его записать система автоматически предлагает произвести обновление расчетов по моделям.

Кнопка Обновить зависимые позволяет выполнить перерасчет зависимых оборотов (плановых показателей). При записи документа такой перерасчет происходит автоматически.

Кнопка Обновить внешние позволяет обновить данные наследуемых оборотов (формируемых извне и отображаемых только информационно). Автоматически такое обновление происходит каждый раз при открытии формы.

Кнопка Установить состояние статей позволяет присвоить всем статьям выбранное состояние.

Кнопка Перезаполнить статьями бюджета очищает все внутренние данные документа и заполняет списком статей. Автоматически в список редактируемых статей вводятся только те статьи, которые не являются зависимыми, рассчитываемыми по моделям или наследуемыми, так как суммы по данным статьям формируются системой. Однако возможно вручную добавить строки с зависимыми или рассчитываемыми по моделям статьями, но они будут отображаться в отдельных строках от рассчитанных автоматически.

Команда Обновить цены из источников меню Действия в правой части закладки «Бюджеты» выполняет пересчет цен и сумм в согласии со способами расчета цен, установленными в финансовых показателях.

Порядок работы с автоматически рассчитываемыми строками бюджета

Следует обратить внимание на расчет различных строк бюджета, ввод которых автоматизирован:

1.       Наследуемые статьи (статьи, у которых указан финансовый показатель, фигурирующий в других бюджета, формируемых раньше по регламенту) – такие строки отображаются уже заполненными. В документе такие цифры не хранятся. Значения для данных строк обновляются при открытии документа, при его перезаполнении, а также по команде «Обновить внешние». В остальные промежутки времени значения данных строк отражаются в том состоянии, в котором они были на момент обновления, даже если их значения изменены другими документами;

2.       Зависимые статьи (статьи, которые получаются в результате расчета зависимостей финансовых показателей) – такие строки недоступны для редактирования. Расчет показателей, зависящих от статей редактируемого бюджета, производится либо при записи/проведении документа, при расчете по моделям, а также по кнопке «Обновить зависимые». В том числе рассчитываются и те статьи, которые относятся к другим бюджетам, но рассчитываются на основании текущего. Статьи, которые относятся к текущему бюджету, но зависят от других бюджетов, отображаются уже заполненными. В документе такие цифры не хранятся. Значения для данных строк обновляются при открытии документа, при его перезаполнении, а также по команде «Обновить внешние». В остальные промежутки времени значения данных строк отражаются в том состоянии, в котором они были на момент обновления, даже если их значения изменены другими документами;

3.       Рассчитываемые по моделям (которые должны рассчитываться по настроенным регламентным моделям) – рассчитываются по кнопке «Расчет по моделям». Если в бюджете присутствуют статьи, которые должны быть рассчитаны по моделям, то их расчет будет произведен. Если в документе есть рассчитываемые по моделям строки, и документ был изменен, но модели не были рассчитаны после этого, то перед закрытием документа система будет предлагать произвести пересчет по моделям. Для отладки функционирования сложных моделей на стадии их разработки можно использовать специальные команды, доступные из меню «Действия», расположенного справа от поля бюджета. К таким командам относятся:

a.       Расчет по моделям (отладка) – будет произведен расчет моделей и после этого будет открыто окно, в котором можно будет увидеть различные первичные и рассчитанные данные, использованные при расчете моделей;

b.       Расчет по моделям (до точки останова) – позволяет при расчете моделей выполнить расчет моделей с приоритетом до указанного, а также для циклических моделей указать предельный номер выполняемого цикла;

c.        Расчет по моделям (после точки останова) – позволяет при расчете моделей выполнить расчет моделей с приоритетом больше указанного, а также для циклических моделей указать начальный номер выполняемого цикла. При этом уже существующие расчеты не будут удалены, таким образом можно отследить появление результирующих данных моделей по шагам;

4.       Рассчитываемые по производственным моделям – расчет таких строк производится по нажатию кнопки «Расчет по моделям» совместно с расчетами по моделям. При этом, в отличие от обычных расчетов по моделям, рассчитываются не статьи, относящиеся к данному бюджету, а статьи, рассчитываемые на основании строк текущего бюджета;

5.       Статьи, рассчитываемые на основании остаточных и рассчитываемых по формулам финансовых показателей – если такие статьи опираются полностью или частично на данные других бюджетов, то они отображаются уже полностью/частично рассчитанными. Если такие статьи опираются на данные текущего бюджета, то имеется два варианта работы, управляемые переключателем «Интерактивный расчет формул» в верхней командной панели документа:

a.       Интерактивный расчет включен – в этом случае такие статьи будут пересчитываться мгновенно при изменении других статей бюджета, влияющих на них. Но при этом возможно снижение производительности;

b.       Интерактивный расчет отключен – в этом случае такие статьи будут пересчитываться только при обновлении зависимых, внешних оборотов, либо всего поля бюджета. Это позволяет повысить производительность работы при редактировании бюджетов. По-умолчанию документ работает именно в таком режиме.

Дополнительные функции редактирования бюджета

Для удобства редактирования бюджета имеется возможность отмены/возврата последних действий пользователя. Для этого в параметрах учета указывается значение «Количество запоминаемых действий редактирования бюджета для отмены». Оно определяет, какой объем памяти будет выделяться для запоминания состояний документа. Увеличение количества запоминаемых операций напрямую зависит от производительности компьютера. Если данное значение больше нуля, то в верхней командной панели становятся доступными команды «Отменить действие» и «Вернуть действие».

Для того, чтобы хранить заметки по документу целиком или отдельным его статьям, или же просматривать заметки, оставленные другими пользователями, следует использовать команду «Заметки», которая доступна из верхней командной панели форм документа и списка, а также из контекстного меню поля редактирования данных бюджета (для конкретной строки бюджета).

Для удобства при работе с большими бюджетами для каждой его строки можно устанавливать состояние (утвержден/отклонен и т.д.). Это полезно в ситуациях, когда бюджет утверждается на протяжении долгого времени, постатейно, для хранения пометок о том, какие статьи уже утверждены. При изменении состояния у самого документа система предлагает установить аналогичное состояние у всех его строк. Если есть неутвержденные строки, система не даст провести документ, если иное не настроено учетной политикой.

Печатные формы

Для документа доступны следующие печатные формы:

·         Бюджет (для вида операции Бюджет и Заявка) – печать бюджета, формируемого данным документом;

·         Бюджет (для вида операции Корректировки) – печать бюджета с учетом корректировок, с их анализом;

·         Корректировки (для вида операции Корректировка) - печать корректировок, введенных документом.
