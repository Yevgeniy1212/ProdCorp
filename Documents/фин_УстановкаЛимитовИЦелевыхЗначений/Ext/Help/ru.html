Документ «Установка лимитов и целевых значений»

Ручная настройка контролируемых показателей производится документами «Установка лимитов и целевых значений» (меню «Контроль» - «Установка лимитов и целевых значений»).

Варианты заполнения реквизитов:

·         «Вид контроля» - определяет характер контроля по установленным границам:

o    Ограничивающее – жесткое ограничение, запрещающее выполнять операции, превышающие установленный лимит. В таком случае система не дает выполнять плановые и фактические операции (проводить документы);

o    Целевое – мягкое ограничение, служащее для сигнализации о превышении лимита в момент превышения и для сравнения показателей с установленными лимитами в отчетах, не более того;

·         «Использование контроля» - область применения контроля в согласии с установленными лимитами:

o    При планировании – в таком случае контролю будут подлежать только операции по планированию бюджетов (в том числе и при автоматической регистрации зависимых оборотов). В этом случае становится доступным для заполнения поле «Контролируемый сценарий», в нем указывается сценарий, к данным которого будут применяться правила контроля;

o    При исполнении – контролю будут подлежать только операции по регистрации фактических данных.

Далее необходимо настроить один из трех вариантов задания лимитов:

·     предварительный бюджет – лимиты задаются списком ограничений по финансовым показателям с указанием аналитики и величины лимита, в табличном поле «Список ограничений»:

Для каждого лимита указывается сумма лимита (предельное значение оборота по выбранной комбинации измерений бюджетирования за период), финансовый показатель, подлежащий контролю, период (который будет отнесен к периоду планирования в зависимости от периодичности сценария, либо к месяцу если речь идет о контроле фактических показателей). Если необходимо, действие лимита ограничивается определенными значениями аналитических разрезов контроля – если значение разреза заполнено, тогда будут контролироваться только обороты по бюджетам с таким же значением данного разреза аналитики;

·     контролирующий сценарий по всем оборотам – в таком случае в качестве лимитов принимаются все плановые показатели определенного сценария, выбранного в поле «Контролирующий сценарий» (например, в том случае, если данные какого-то сценария представляют собой контрольный бюджет или нормативную смету затрат и т.д.). В таком в документе настраивается использование данных сценария для контроля прочих плановых и фактических данных:

Указывается интервал плановых данных сценария, которые будут использоваться в качестве лимитов. Также настраивается допустимое отклонение от данных сценария – в поле «Вид допустимого отклонения» указывается вид отклонения – либо абсолютное (тогда в поле «Отклонение» задается абсолютная сумма возможного отклонения по каждой статье), либо относительное (тогда в поле «Отклонение» задается процент возможного отклонения по каждой статье). Указывается необходимость контроля по определенным значениям разрезов аналитики. Если включен контроль по какому-либо разрезу, действие лимита ограничивается определенными значениями аналитических разрезов контроля – теми, которые имеются в плановых данных сценария – и по прочим значениям указанных разрезов контроль производиться не будет. Если же контроль по измерению не включен, это означает что сформированные лимиты относятся ко всем оборотам независимо от значения аналогичного разреза аналитики;

·     контролирующий сценарий по выборочным оборотам - в таком случае в качестве лимитов принимаются плановые показатели определенного сценария, выбранного в поле «Контролирующий сценарий» (например, в том случае, если данные какого-то сценария представляют собой контрольный бюджет или нормативную смету затрат и т.д.), однако используемые в качестве лимитов показатели сценария ограничиваются заданным списком. В таком случае настраивается список используемых показателей и их использование для контроля прочих плановых и фактических данных:

Для каждой строки списка указывается финансовый показатель, обороты по которому берутся из данных сценария, период. Если необходимо, действие лимита ограничивается определенными значениями аналитических разрезов контроля – если значение разреза заполнено, тогда из данных сценария в качестве лимитов будут приняты только показатели с таким же значением данного разреза аналитики

Мастер заполнения документа "Установка лимитов и целевых значений"

     В документе "Установка лимитов и целевых значений" с видами ограничения оборотов "Предварительный бюджет" и "Контролирующий сценарий (выборочно)" имеется возможность автоматического заполнения строк документа при помощи мастера. Мастер открывается из формы документа, из пункта меню "Добавить" командной панели списка ограничений. В данном меню есть два пункта: "Заполнить" (в этом случае табличная часть будет полностью перезаполнена по результатам работы мастера) и "Добавить" (в этом случае табличная часть будет дополнена по результатам работы мастера).

 Способы установки контролируемых параметров

Предварительный бюджет

Данный вид ограничения оборотов задается для определенного сценария планирования. Пользователем указывается вручную список финансовых показателей, по которым будут контролироваться данные. Указывается также сумма по финансовому показателю, период действия контроля, аналитика по подразделению, проекту, контрагенту и номенклатуре. Период действия всегда равен одному периоду планирования, согласно выбранному сценарию, поэтому фактически пользователь устанавливает только дату начала периода планирования, а конец периода определяется длиной периода планирования.

Контролю подлежат только суммовые обороты, количественные обороты не контролируются.

Если какая-либо аналитика (подразделение, проект, контрагент, номенклатура) не будет задана, то контролю подлежат итоговые обороты по такой аналитике.

Контролирующий сценарий (по всем оборотам)

При этом варианте контроля обороты одного сценария являются контролируемыми значениями для оборотов другого сценария. Контролируемый и контролирующий сценарий в общем случае могут быть разной периодичности, поэтому при установке периода действия контроля выбирается наибольший период из периодичности этих сценариев. Начало и конец периода устанавливается пользователем.

Важно!!! Для контроля используются данные каждой статьи сценария. Можно дополнительно указать по какой аналитике будет осуществляться контроль (подразделение, проект, номенклатура, контрагенты).

В отличие от контроля по бюджету, где контролируемые значения задаются жестко фиксированными суммами, здесь значения оборотов по статьям контролирующего сценария указываются в относительном выражении. То есть устанавливаются допустимые пределы превышения оборотов контролируемого сценария над оборотами контролирующего. Отклонение задается в абсолютном  или в относительном выражении. Абсолютные отклонения задаются в валюте контролируемого сценария.

Например, для контролирующего сценария по аналитике подразделение задано абсолютное отклонение равное 140. Это означает, что при планировании оборотов по контролируемому сценарию превышение оборотов по любой статье этого сценария не должно превышать более, чем на 140 единиц, обороты по такой же статье контролирующего сценария.

Важно!!! Значения оборотов контролирующего сценария фиксируются при определении контролируемых значений. И превышение анализируется относительно зафиксированных значений, то есть значения контролируемых параметров не вычисляются каждый раз при планировании операции. Отклонение одинаково для всех статей сценария.

Контролирующий сценарий по выборочным статьям оборотов

Способ задания параметров аналогичен предыдущему способу, однако рассмотрению подлежат не все статьи сценария, а указанные пользователем. Для каждой статьи указывается период действия контроля, вид отклонения, значение отклонения, настраиваются значения аналитик бюджетирования.
