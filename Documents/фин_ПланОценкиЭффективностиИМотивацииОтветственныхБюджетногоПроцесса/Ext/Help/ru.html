План оценки эффективности и мотивации ответственных бюджетного процесса

Документ "План оценки эффективности и мотивации ответственных бюджетного процесса" позволяет настроить правила, по которым будет оцениваться эффективность работы ЦФО их руководителей, в унифицированном и сопоставимом виде.

Документ позволяет настроить правила перевода результатов достижения бюджетных показателей ЦФО в унифицированные (сопоставимые с результатами других ЦФО) коэффициенты. Расчет эффективности производится на основании отклонений фактических данных от плановых, и последующей оценке величины данных отклонений.

Эти правила настраиваются в табличной части "Расчет коэффициентов эффективности". Для каждого ЦФО можно настроить любое количество правил расчета.

В строке табличной части задаются следующие параметры:

  • Управленческое подразделение (ЦФО), по которому настраивается правило расчета;
  • Ключевой показатель - финансовый показатель, данные которого будут использоваться для оценки эффективности:
  • Цель - цель ответственного лица по достижению планового значения показателя (минимизировать, максимизировать или достигнуть без отклонений). позволяет определить, какой результат исполнения бюджета считать положительным - если целью является минимизировать расход, то положительным будет считаться отклонению в меньшую от плана сторону, если целью стоит максимизировать доход, то положительным будет считаться отклонение в большую сторону. Если выбрана цель "Точное достижение", то любое отклонение будет считаться отрицательным;
  • Данные - какие данные (суммовые или количественные) будут анализироваться;
  • Ограничить подразделением - если данный признак не установлен, то будут анализироваться все данные финансового показателя, независимо от наличия разреза учета по подразделениям и от того, по каким подразделениям введены плановые и фактические данные по нему. Если признак установлен, то будут анализироваться только те данные, которые введены по подразделению, для которого настраивается правило;
  • Используемая величина - определяет, что будет взято за величину коэффициента эффективности по данному правилу (далее используются следующие понятия: План - плановая величина; Факт - Фактическая величина; Отклонение - разница между планом и фактом, при этом знак, с которым эта разница будет использоваться, определяется целью, указанной в строке правила, т.е. если целью является минимизация и факт меньше плана, тогда разница будет взята со знаком "плюс", а если факт больше плана, то со знаком минус "минус"):
    • Относительное исполнение (%) - процент исполнения плана ([Факт/План]*100);
    • Относительное отклонение от плана (%) - процент отклонения от плана ([Отклонение/План]*100);
    • Абсолютная величина исполнения (Факт);
    • Абсолютное отклонение от плана (Отклонение);
    • Формула - пользователь самостоятельно может настроить формулу в поле "Формула";
  • Формула - вводится, если в качестве используемой величины выбран расчет по формуле. Пользователь может ввести в текстовом виде формулу, используя арифметические и логические выражения встроенного языка 1С:Предприятия. При этом можно оперировать переменными План и Факт. На их место при расчете будут подставлены соответствующие значения;
  • Удельный вес в итоговом коэффициенте - доля значимости данного правила. На нее будет умножаться результат расчета данного коэффициента. По-умолчанию устанавливается значение 1. Установка данного реквизита может понадобиться в том случае, если результат нужно просто умножить на какую-то величину, либо если по подразделению настроено несколько правил и они должны внести определенный вклад в расчет итогового коэффициента пропорционально своей значимости. К примеру, в качестве коэффициента эффективности планируется использовать процент исполнения плана. Его целевое значение, как правило равно 100. Если по одному подразделению оценивается исполнение только по одной статье бюджета, то для него сумма результатов будет равна 100 (при точном выполнении бюджета). Если же у другого подразделения настроено два и более правила, для учета исполнения двух или более статей, то суммарный процент исполнения будет складываться по каждому правилу и будет равен 200 и более при прочих равных условиях. В итоге получится несопоставимая между двумя подразделениями оценка эффективности. Поэтому для второго подразделения мы можем настроить удельный вес для каждого правила, например установив доля 0.5 в каждом из двух правил, или другое соотношение коэффициентов (в зависимости от степени вклада того или иного показателя).

Расчет результатов в каждом нужном периоде будет осуществляться документом "Расчет коэффициентов эффективности работы ответственных бюджетного процесса".

Также, помимо правил расчета эффективности, в данном документе есть возможность настроить правила материального стимулирования ответственных за ЦФО. Для этого используется табличная часть "Схема мотивации". Входными данными для расчета премий и удержаний (штрафов) будут коэффициенты эффективности, рассчитанные на основании правил, описанных выше.

В строке данной табличной части настраиваются следующие поля:

  • Управленческое подразделение (ЦФО), по которому настраивается правило мотивации;
  • Формула - формула расчета величины, используемой для назначения премии/удержания. Пользователь может ввести в текстовом виде формулу, используя арифметические и логические выражения встроенного языка 1С:Предприятия. При этом можно оперировать переменной Коэффициент. На ее место при расчете будет подставлено соответствующее значение. Данная формула должна вернуть положительное значение, если требуется выдать премию, и отрицательное, если требуется произвести удержание (штраф). Значение должно соответствовать размеру премии/удержания в кадровом контуре, в зависимости от способа расчета указанного вида расчета;
  • Премировать при положительном коэффициенте - определяет необходимость начислить премию, если результат расчета по формуле больше нуля;
  • Вид расчета премии - вид расчета начислений информационной базы, используемый для отражения премии;
  • Минимальный размер для расчета премии - минимальное значение результата расчета по формуле, при котором начинает начисляться премия;
  • Удерживать при отрицательном коэффициенте - необходимость регистрации удержания (штрафа) при отрицательной величине расчета по формуле;
  • Вид расчета удержания - вид расчета удержаний информационной базы, используемый для отражения удержания;
  • Начальный размер для расчета удержания - начальный размер отрицательного результата расчета, начиная с которого будет регистрироваться удержание (Штраф).

Расчет премий и удержаний производится после расчета коэффициентов эффективности за соответствующий период. Осуществляется в документе "Расчет по плану мотивации ответственных бюджетного процесса". Расчет коэффициентов производится в каждом интервале бюджетного периода, в котором это необходимо. Шаг бюджетного периода будет определяться сценарием. Начисление премий и удержаний рассчитывается за весь бюджетный период. Например, используется сценарий квартальный помесячно. В таком случае коэффициенты эффективности будут рассчитаны в каждом месяце этого квартала. Расчет премий и удержаний будет производиться за указанный интервал (это может быть отдельный расчет за каждый месяц, расчет за весь квартал, полугодие или год), и для расчета размера премий/удержаний по формулам из табличной части "Схема мотивации" будет использовано суммированное значение коэффициентов за все месяца указанного интервала. Таким образом следует учесть данный нюанс в настройке формул расчета премий/удержаний при использовании расчета премий и удержаний за несколько месяцев сразу.
