Отчет "Расходы периода (XML)"

Отчет "Расходы периода" предназначен для составления отчетности по всем расходам организации, которые согласно стандарту бухгалтерского учета № 7 «Учет товарно-материальных запасов», утвержденному постановлением Национальной Комиссии по бухгалтерскому учету от 13 ноября 1996 года № 3, зарегистрированному в Министерстве юстиции Республики Казахстан 30 декабря 1996 года № 239, не включаются в себестоимость продукции (товары, работы, услуги) и признаются расходами периода.

Отчет содержит дебетовые обороты счетов: 7210 (первый раздел отчета), 7110 (второй раздел отчета), 7310 (третий раздел отчета). Для каждой статьи затрат, использующейся на данных счетах, нужно указать вид расхода периода в регистре сведений "МОП: соответствия статей затрат". Классификатор видов расходов периода заполняется по кнопке "Заполнить" на форме его списка.
При необходимости (если проводки по одной статье затрат, например, "социальный налог" выполняются одновременно по двум счетам, например по "Административным расходам" и "Расходам по реализации"), можно указать одной статье затрат два вида расхода периода, в полях "Вид расхода периода" и "Вид расхода периода дополнительный".

Если для статьи затрат, использованной в оборотах указанных счетов, не указан вид расхода периода, будет выдано соответсвующее предупреждение: "Обнаружены статьи затрат, у которых не указан вид расхода периода, суммы по ним не попадут в отчет..." с указанием статей затрат.
Если для статьи затрат, использованной в оборотах конкретного счета, не указан вид расходов, относящийся к соответствующему разделу отчета (например, для счета 7210 - первый раздел, 7110 - второй раздел, 7310 - третий раздел), будет выдано соответствующее предупреждение: "Обнаружены статьи затрат, которые не попали в разделы отчета из-за несоответствующего указания видов расходов периода..." с указанием статей затрат и разделов, из которых должен быть указан вид расходов.

Обороты счета 7310 попадают в строки "резидент" / "нерезидент" (см. таблицу отчета) в зависимости от КБЕ контрагента, указанного в качестве второго субконто на этом счете. Если первый символ КБЕ равен "1", сумма попадает в строку "резидент", иначе - в строку "нерезидент", если субконто "Контрагент" незаполнено в проводке, сумма попадает в строку "резидент".

 Общая информация о работе с отчетами

 Отчет имеет две формы.

В первой, основной форме пользователю нужно установить Организацию и период отчета.

 

После нажатия «ОК» открывается форма отчета, верхняя часть которой используется для управления формированием, а в нижней части расположен сам бланк отчета.

 

Работа с отчетом

 

Для работы необходимо установить значение следующих реквизитов в форме отчета:

 

«Вид формы»   -   вид результирующей формы - «Основная» или «Корректировочная», значение выбирается из выпадающего списка

 

«Код налоговой инспекции»   -   код налоговой инспекции Организации

  

В диалоговом окне формы отчета предусмотрен набор кнопок, соответствующий списку

возможных действий:

 

Кнопка «Выгрузить в XML» -  при помощи этой кнопки осуществляется выгрузка информации в формате XML для клиентского приложения ИС МОП.

                                                                         

Кнопка «Заполнить»  -  по данной кнопке происходит автоматическое заполнение отчета данными из бухгалтерского учета. При желании пользователь может отредактировать или заполнить данные самостоятельно.

 

Кнопка «Очистить» -  очищает бланк отчета

 

Кнопки добавления, удаления строк – пользователь может добавить новую строку (и заполнить ее самостоятельно) или удалить любую выделенную

 

Кнопки последовательного перехода по страницам – пользователь может перейти на следующую / предыдущую страницу отчета; текущая страница и полное количество страниц отображаются между этими кнопками

 

Кнопка «Печать» - позволяет выполнить предварительный просмотр, либо сразу распечатать отчет

 

Кнопка произвольного перехода к странице – пользователь может перейти на любую страницу отчета.

 

Кнопка вызова справки – пользователь может просмотреть текущую справку, либо перейти к общей информации о формах.

  

Кроме этого, в процессе работы с отчетом пользователь может самостоятельно редактировать данные, представленные в бланке отчета. По двойному щелчку на ячейке открывается элемент (карточка счета, контрагента или товара, и т.п.), после редактирования которого обновляются все связанные поля (адрес контрагента, код ТН ВЭД товара, и т.п.). При двойном щелчке на ячейке с обычным текстом или числом (например, в поле "Сумма НДС"), открывается окно для прямого редактирования значения.

 

Все основные механизмы работы с отчетом (печать, форматирование данных, выгрузка в

XML-формате и т.д.) заимствованы из  регламентированных отчетов типовой конфигурации
