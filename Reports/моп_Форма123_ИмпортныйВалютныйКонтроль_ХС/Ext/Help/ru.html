Отчет "Импортный валютный контроль (XML)"

Отчет "Импортный валютный контроль" предназначен для составления отчетности по купленным за пределами Казахстана ТМЗ, ОС и НМА.

В отчет попадают все строки регистра накопления "Мониторинг импорта" который заполняется документом "Движения мониторинга". В отчет попадают данные по операциям приобретения товаров из документов, с установленным видом операции "Импорт", исключение составляет операция по импорту услуг.

Импорт услуг

Операция импорта услуг оформляется с видом "Покупка". Для того, чтобы операции по импорту услуг попали в отчет, необходимо сформировать документ "Корректировка записей регистров" и внести запись в регистр накопления "Взаиморасчеты по НДС, подлежащему уплате за нерезидента". Для данной записи вид движения должен быть "приход". В реквизите "Номенклатура" указывается номенклатура, по которой происходит начисление НДС за нерезидента, в реквизите "Сумма НДС" указывается сумма НДС уплачиваемого за нерезидента, а в реквизите"Документ поступления" указывается ссылка на документ "Поступление ТМЗ и услуг" (или "Поступление НМА"), которым оформили поступление.

"ГТД по импорту" и "Заявление о вводе товаров и уплате косвенных налогов"

Некоторые колонки отчета (см. ниже) заполняются по данным документов "ГТД по импорту" и "Заявление о вводе товаров и уплате косвенных налогов". Предполагается, что если выполняется импорт товаров из страны-участника таможенного союза, то оформляется документ "Заявление о вводе товаров и уплате косвенных налогов" где в табличных частях "ТМЗ" и "ОС" в реквизите "Документ-основание" указывается документ, в котором был оформлен импорт. Если выполняется импорт товаров не из страны-участника таможенного союза, то оформляется документ "ГТД по импорту" где в табличных частях "ТМЗ" и "Основные средства" в реквизите "Документ партии" указывается документ, в котором был оформлен импорт. Таким образом реализуется связь с документами поступления товаров (накладными).

Расшифровка некоторых колонок отчета:

  • Номер строки: двойной щелчок откроет документ поступления;
  • Код ТН ВЭД - код ТНВЭД, заданный, для соответствующего объекта мониторинга, в регистре "Объекты мониторинга". Двойной щелчок откроет карточку товара, откуда можно перейти к регистру "Объекты мониторинга";
  • Пин-код - персональный идентификационныq номер–код бензина (за исключением авиационного), дизельного топлива и мазута, в конфигурации "Бухгалтерия для Казахстана", заполняется из регистра "Объекты мониторинга", если установлено дополнение "1С-Рейтинг: Нефтебаза", заполняется из документов поступления ГСМ (Горюче-смазочных материалов);
  • Дата ТД (Заявления) – дата документа "ГТД по импорту", "ГСМ: ГТД по импорту" или "Заявление о ввозе товаров и уплате косвенных налогов";
  • Номер ТД (Заявления) – номер декларации (ГТД) в документе "ГТД по импорту", "ГСМ: ГТД по импорту" или "Заявление о ввозе товаров и уплате косвенных налогов";
  • Цена, Количество, Сумма (без налогов) – суммы в валюте из табличной части документа поступления;
  • Валюта контракта – Валюта операций из документа поступления;
  • Курс тенге – Курс валюты, указанный в документе поступления;
  • Фактически уплачено НДС – сумма НДС по данной номенклатуре из документа "ГТД по импорту", "ГСМ: ГТД по импорту", "Заявление о ввозе товаров и уплате косвенных налогов" или из реквизита "Сумма НДС" регистра "Взаиморасчеты по НДС, подлежащему уплате за нерезидента";
  • Акцизы  – сумма акциза, заполняется автоматически, если дополнение встроено в конфигурацию "1С-Рейтинг: Нефтебаза редакция 1.5" релиз 1.5.14.3 и выше. Автоматическое заполнение выполняется по данным документов "ГСМ: ГТД по импорту" введенных на основании документов поступления ГСМ. Сумма акциза берется из графы "Акциз" табличной части "ГСМ" документа "ГСМ: ГТД по импорту";
  • Таможенные пошлины и сборы – это сумма сбора по текущей номенклатуре + сумма сбора в валюте по текущей номенклатуре, пересчитанная в тенге + доля общей суммы пошлины (пропорция по сумме поступления), касающаяся текущей номенклатуры. Если пошлина указана в валюте, то она пересчитывается в тенге. Все эти суммы берутся из документов "ГТД по импорту", "ГСМ: ГТД по импорту" или "Заявление о ввозе товаров и уплате косвенных налогов";
  • Место отгрузки – код страны поставщика.

 

 

Общая информация о работе с отчетами

 

Отчет имеет две формы.

В первой, основной форме пользователю нужно установить Организацию и период отчета.

 

После нажатия «ОК» открывается форма отчета, верхняя часть которой используется для управления формированием, а в нижней части расположен сам бланк отчета.

 

Работа с отчетом

 

Для работы необходимо установить значение следующих реквизитов в форме отчета:

 

«Вид формы»   -   вид результирующей формы, значение выбирается из выпадающего списка

 

«Код налоговой инспекции»   -   код налоговой инспекции Организации

  

В диалоговом окне формы отчета предусмотрен набор кнопок, соответствующий списку

возможных действий:

 

Кнопка «Выгрузить в XML» -  при помощи этой кнопки осуществляется выгрузка информации в формате XML для клиентского приложения ИС СОНО.

 

Кнопка «Выгрузить в Excel»  при помощи этой кнопки осуществляется выгрузка информации в формате MS Excel для клиентского приложения ИС СОНО

                                                                         

Кнопка «Заполнить»  -  по данной кнопке происходит автоматическое заполнение отчета данными из бухгалтерского учета. При желании пользователь может отредактировать или заполнить данные самостоятельно.

 

Кнопка «Очистить» -  очищает бланк отчета

 

Кнопки добавления, удаления строк – пользователь может добавить новую строку (и заполнить ее самостоятельно) или удалить любую выделенную

 

Кнопки последовательного перехода по страницам – пользователь может перейти на следующую / предыдущую страницу отчета; текущая страница и полное количество страниц отображаются между этими кнопками

 

Кнопка «Печать» - позволяет выполнить предварительный просмотр, либо сразу распечатать отчет

 

Кнопка произвольного перехода к странице – пользователь может перейти на любую страницу отчета.

 

Кнопка вызова справки – пользователь может просмотреть текущую справку, либо перейти к общей информации о формах.

  

Кроме этого, в процессе работы с отчетом пользователь может самостоятельно редактировать данные, представленные в бланке отчета. По двойному щелчку на ячейке открывается элемент (карточка счета, контрагента или товара, и т.п.), после редактирования которого обновляются все связанные поля (адрес контрагента, код ТН ВЭД товара, и т.п.). При двойном щелчке на ячейке с обычным текстом или числом (например, в поле "Сумма НДС"), открывается окно для прямого редактирования значения.

 

Все основные механизмы работы с отчетом (печать, форматирование данных, выгрузка в

XML-формате и т.д.) заимствованы из  регламентированных отчетов типовой конфигурации

"Бухгалтерия для Казахстана"
