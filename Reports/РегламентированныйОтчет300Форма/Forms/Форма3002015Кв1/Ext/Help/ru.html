Отчет «Декларация по налогу на добавленную стоимость, форма 300.00 (XML)» за 2015 год.

Отчет «Декларация по налогу на добавленную стоимость, форма 300.00 (XML)» предназначен для исчисления сумм налога на добавленную стоимость в соответствии с разделом 8 Налогового кодекса и статьями 2, 12, 24, 25, 28, 32, 34, 35, 40, 48, 49 Закона Республики Казахстан «О введении в действие Кодекса Республики Казахстан «О налогах и других обязательных платежах в бюджет» (Налоговый кодекс)» (далее - Закон о введении).

Если в настройках параметров учета (меню Предприятие\Настройка параметров учета) установлен признак поддержки работы со структурными подразделениями, то при открытии отчета необходимо указать налогоплательщика и налоговый комитет, в который будет предоставлен отчет. Если работы со структурными подразделениями в налоговом учете не поддерживается, то при открытии отчета требуется выбрать только налогоплательщика, налоговый комитет будет определен по реквизиту «Налоговый комитет» из справочника «Организации».

Для проверки правильности составления Декларации предназначен отчет «Список / кросс-таблица», позволяющий в произвольных разрезах проанализировать корректность формирования регламентированного отчета.

Внимание! Автоматическое заполнение предусмотрено только для строк, перечисленных ниже по тексту, все прочие строки формы в случае наличия данных, отражаемых в них, необходимо заполнить «вручную».

Раздел. Сумма НДС, относимого в зачет

Заполнение строк 300.00.013, 300.00.015, 300.00.016I, 300.00.016II, 300.00.020 осуществляется расчетным путем. Для того, чтобы в вышеуказанных строках корректно отобразились данные необходимо при формировании соответствующих документов поступления правильно указывать вид оборота и вид операции поступления. Так, при использовании пропорционального метода отнесения НДС в зачет в документах разрешается использовать только вид оборота «общий». При раздельном методе отнесения НДС в зачет допускается в зависимости от смысла отражаемой хозяйственной операции использовать один из трех видов оборота («общий», «облагаемый» либо «освобожденный»).

Внимание! При оформлении документов следует четко контролировать, чтобы вид оборота, указанный в табличной части документа, полностью соответствовал виду оборота, указанному в форме элемента выбранного в этом же документе вида поступления. В случае расхождения видов оборота, такие данные также не будут учтены при формировании соответствующих строк Декларации.

    • При использовании пропорционального метода отнесения НДС в зачет при заполнении строк 300.00.013, 300.00.016, 300.00.016I, 300.00.016II и 300.00.020 указывается общая сумма НДС, относимого в зачет.
    • При использовании раздельного метода отнесения НДС в зачет при заполнении строк 300.00.013, 300.00.016, 300.00.016I, 300.00.016II, 300.00.020 общая сумма НДС, относимого в зачет, определяется следующим образом: сумма НДС, которая прошла по виду оборота «облагаемый» учитывается полностью. Сумма НДС по виду оборота "освобожденный" не учитывается в полном объеме, а сумма НДС, которая была зарегистрирована по виду оборота «общий», берется в зачет по удельному весу облагаемого оборота в общем обороте по реализации.

Так в раздел «Оборот по реализации, облагаемый по нулевой ставке» приложения 300.01 попадут только данные о тех оборотах по реализации, которые прошли по ставке НДС 0%, и у которых вид операции реализации ссылается на код строки 300.01.001, 300.01.001.I, 300.01.001.II, 300.01.001.III, 300.01.002, 300.01.003 или 300.01.004.

Строка 300.01.001 включает в себя данные, отраженные на коду строки 300.01.001, а также данные отраженные по коду строк 300.01.001.I, 300.01.001.II и 300.01.001.III.

300.02 ОБОРОТ ПО РЕАЛИЗАЦИИ ТОВАРОВ, РАБОТ, УСЛУГ, И ИМПОРТ, ОСВОБОЖДЕННЫЙ ОТ НДС (Приложение № 2 к Декларации)

 

Раздел. Оборот по реализации, освобожденный от НДС

Заполнение соответствующих строк раздела «Оборот по реализации, освобожденный от НДС» приложения 300.02 осуществляется на основании данных о видах реализации, указанных в регистре накопления «Налог на добавленную стоимость», а также ставках НДС. Так в соответствующие строки приложения 300.02 попадут данные только о тех оборотах по реализации, которые прошли по ставке НДС - «Без НДС» с установленным признаком «Для освобожденного оборота». Этот признак может быть установлен в справочнике «Ставки НДС» (меню Предприятие – Налоги – Налог на добавленную стоимость – Ставки НДС). Кроме этого, вид операции реализации должен ссылаться на ту или иную строку раздела «Оборот по реализации, освобожденный от НДС» приложения 300.02

Раздел. Импорт, освобожденный от НДС

Для корректного формирования раздела «Импорт, освобожденный от НДС» приложения 300.02 необходимо при создании документов, отражающих поступление ТМЗ и услуг (ГТД по импорту), указывать ставку НДС - «Без НДС», у которой установлен признак «Для освобожденного оборота». Кроме этого, в табличной части документов поступления необходимо выбирать вид поступления, который ссылается на ту или иную строку раздела «Импорт, освобожденный от НДС» приложения 300.02.

300.03 ИМПОРТ ТОВАРОВ, ПО КОТОРЫМ ИЗМЕНЕН СРОК УПЛАТЫ НДС (Приложение № 3 к Декларации)

 

Учет НДС с измененным сроком уплаты в конфигурации «1С:Бухгалтерия 8 для Казахстана» организован с помощью регистра накопления «Взаиморасчеты с бюджетом по НДС с измененным сроком уплаты». Для корректного отображения в соответствующих приложениях Декларации данного вида НДС в конфигурации необходимо придерживаться некоторых правил заполнения документов, с помощью которых организован учет этого вида НДС:

При заполнении документа «ГТД по импорту» в табличной части закладки «Разделы ГТД» необходимо указать в графе «Вид НДС» - «НДС с измененным сроком уплаты», в графе «Вид поступления» - указать вид поступления, который ссылается либо на строку 300.03.001, либо на строку 300.03.002.

Кроме этого, обязательному заполнению подлежит графа «Срок уплаты НДС», в которой указывается конечная граница срока перечисления НДС в бюджет по данной операции. При проведении документа ГТД по импорту данные отражаются в регистре накопления «Взаиморасчеты с бюджетом по НДС с измененным сроком уплаты» с видом движения «приход», таким образом, происходит «начисление» задолженности перед бюджетом по данному виду налога в регистре.

Операция перечисления НДС с измененным сроком уплаты в бюджет оформляется с помощью документа «Платежное поручение исходящее» с видом операции «Перечисление в бюджет НДС с измененным сроком уплаты» (меню Банк и касса – Платежное поручение исходящее). При оформлении этого документа на закладке «Перечисление НДС с измененным сроком уплаты» в графе «Вид операции поступления» указывается вид поступления, который также ссылается либо на строку 300.03.001, либо на строку 300.03.002. При проведении документа данные отражаются в регистре накопления «Взаиморасчеты с бюджетом по НДС с измененным сроком уплаты» с видом движения «расход», т.е. происходит «погашение» задолженности перед бюджетом по данному виду налога в регистре. Кроме этого, формируется движение в регистр накопления НДС к возмещению, т.к. только в момент оплаты данный вид НДС принимается к зачету.

Часть задолженности перед бюджетом, которая не была погашена соответствующими банковскими платежами в конце периода, может быть погашена путем зачета соответствующей регламентной операцией «Зачет НДС к возмещению» документа «Закрытие месяца».

 

300.04 ИМПОРТ ТОВАРОВ, НДС ПО КОТОРЫМ УПЛАЧИВАЕТСЯ МЕТОДОМ ЗАЧЕТА (Приложение № 4 к Декларации)

В формировании данного приложения используются записи регистра накопления «НДС к возмещению» с видом налога «НДС, уплачиваемый методом зачета». Документами конфигурации, которые могут отразить движения по данному виду налога в вышеуказанный регистр накопления, являются: «ГТД по импорту» и «Корректировка записей регистров». При формировании документа «ГТД по импорту» в данном случае следует придерживаться ряда правил:

  • При заполнении закладки «Разделы ГТД» в графе «Вид НДС» в обязательном порядке указывается значение – «НДС, уплачиваемый методом зачета»;
  • При заполнении закладки «Разделы ГТД» в графе «Вид операции поступления» указывается тот вид поступления, который ссылается на ту или иную строку приложения 300.04.

 

300.05 РАБОТЫ, УСЛУГИ, ПРИОБРЕТЕННЫЕ У НЕРЕЗИДЕНТА (Приложение № 5 к Декларации)

Учет НДС за нерезидента в конфигурации «1С:Бухгалтерия 8 для Казахстана» ведется «вручную» на регистре накопления «Взаиморасчеты с бюджетом по НДС, подлежащему уплате за нерезидента».

300.06 КОРРЕКТИРОВКА РАЗМЕРА ОБЛАГАЕМОГО И ОСВОБОЖДЕННОГО ОБОРОТОВ И СУММЫ НДС, ОТНЕСЕННОГО В ЗАЧЕТ(Приложение № 6 к Декларации)

При заполнении раздела «Корректировка размера облагаемого и освобожденного оборотов» приложения 300.06 следует учитывать, что данные для заполнения берутся из регистра накопления «Корректировка НДС к возмещению»при условии, что вид операции корректировки ссылается на соответствующий код строки данного приложения.

Таким образом заполнение приложения 300.06 нужно производить после заполнения строки 300.00.001, приложения 300.01, строки 300.00.004 и приложения 300.02.

 

300.07 РЕЕСТР СЧЕТОВ-ФАКТУР ПО РЕАЛИЗОВАННЫМ ТОВАРАМ, РАБОТАМ, УСЛУГАМ В ТЕЧЕНИЕ ОТЧЕТНОГО НАЛОГОВОГО ПЕРИОДА (Приложение № 7 к Декларации)


В приложении отображаются все счета-фактуры, по которым соответствующими документами был зарегистрирован оборот по реализации за период составления отчетности (вне зависимости от даты регистрации счета-фактуры), выписанные по реализации на территорию РК (в качестве страны назначения в счете-фактуре  указана Республика Казахстан - код страны 398).

При этом графа J «Сумма начисленного НДС» складывается из суммы НДС по документу, на основании которого был зарегистрирован счет-фактура. Графа J подлежит обязательному заполнению в случае, если по соотвествующей строке заполнена графа I.

Отчет может быть сформирован в одном из двух режимов: на основании первичных документов или на основании данных регистра "Сведения о счетах-фактурах выданных". Вариант формирования отчета на основании данных регистра позволяет заполнить реестр для контрагентов - участников договора совместной деятельности.

300.08 РЕЕСТР СЧЕТОВ-ФАКТУР (ДОКУМЕНТОВ НА ВЫПУСК ТОВАРОВ ИЗ ГОСМАТРЕЗЕРВА) ПО ПРИОБРЕТЕННЫМ ТОВАРАМ, РАБОТАМ, УСЛУГАМ В ТЕЧЕНИЕ ОТЧЕТНОГО НАЛОГОВОГО ПЕРИОДА (Приложение № 8 к Декларации)


В реестре счетов-фактур отображаются счета-фактуры, у которых дата выписки попадает в отчетный период. В случае если сумма НДС, указанного в счете-фактуре заполнена, а сумма НДС подлежащего отнесению в зачет равна 0, то выдается предупреждающее сообщение пользователю. При этом графа J «Сумма НДС, подлежащего отнесению в зачет» складывается из суммы НДС, указанного в документе, на основании которого зарегистрирован счет-фактура. При этом если документ-основание был заполнен до начала налогового периода, то в графе J «Сумма НДС, подлежащего отнесению в зачет» отображается 0. Для таких строк, в которых сумма НДС к зачету равна 0 - необходимо проверить даты документов поступления.

Отчет может быть сформирован в одном из двух режимов: на основании первичных документов или на основании данных регистра "Сведения о счетах-фактурах полученных". Вариант формирования отчета на основании данных регистра позволяет заполнить реестр для контрагентов - участников договора совместной деятельности.
