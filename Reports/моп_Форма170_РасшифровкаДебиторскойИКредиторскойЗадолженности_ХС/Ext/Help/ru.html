Отчет "Расшифровка дебиторской и кредиторской задолженности (XML)"

Отчет "Расшифровка дебиторской и кредиторской задолженности" предназначен для составления отчетности по задолженностям.

Алгоритм заполнения реализован исходя из предоположения что цель отчета - выявить задолженность, которая образовавшись, не подлежит погашению уже длительное время (1 год и более).

Согласно правилам заполнения:
"В форме 1.7. «Расшифровка дебиторской и кредиторской задолженности» указываются  суммы  дебиторской (кредиторской) задолженности, образовавшейся свыше одного года с момента последней операции по данной задолженности."

 

В отчет попадает сальдо счетов расчетов с контрагентами на дату годом ранее даты начала отчета (т.к. рассматривается только задолженность сроком свыше 1 года). Т.е. если отчет формируется за 1 квартал 2009 года, то сумма задолженности будет анализироваться на дату 31 декабря 2007 года включительно и ранее. Сальдо активных счетов образует дебиторскую задолженность, пассивных и активно-пассивных – кредиторскую.

 В зависимости от периода образования задолженности устанавливается ее срок: «Свыше 1го года», «Свыше 2х лет», «Свыше 3х лет», «Свыше 4х лет», «Свыше 5ти лет» (например, если первое незакрытое сальдо образовалось ранее чем за 2 года до даты начала отчета, то срок будет: «Свыше 3х лет»).

 

Работа с отчетом

 

Отчет имеет две формы.

В первой, основной форме пользователю необходимо установить Организацию и период отчета.

 

После нажатия «ОК» открывается форма отчета, верхняя часть которой используется для управления формированием, а в нижней части расположен сам бланк отчета.

 

В диалоговом окне формы отчета предусмотрен набор кнопок, соответствующий списку

возможных действий:

 

Кнопка  «Форма навигации» - открывает панель навигации между приложениями формы.

 

 

 

При выборе приложения, открывается его форма.

 

 

На форме приложения расположены дополнительные элементы управления формированием отчета: для приложения "Расшифровка дебиторской задолженности" доступно поле "Счета учета дебиторской задолженности", а для приложения "Расшифровка кредиторской задолженности" доступно поле "Счета учета кредиторской задолженности" где задается перечень счетов, по которым будет вычисляться задолженность. При этом, если выбран счет-группа, то задолженность будет получена для всех его субсчетов.

 

Кнопка «Выгрузить в XML» -  при помощи этой кнопки осуществляется выгрузка основной формы и приложений в формате XML для клиентского приложения ИС СОНО

 

Кнопка «Выгрузить в Excel 1.7.01»  при помощи этой кнопки осуществляется выгрузка приложения 1.7.01 "Расшифровка дебиторской задолженности" в формате MS Excel для клиентского приложения ИС СОНО

 

Кнопка «Выгрузить в Excel 1.7.02»  при помощи этой кнопки осуществляется выгрузка приложения 1.7.02 "Расшифровка кредиторской задолженности" в формате MS Excel для клиентского приложения ИС СОНО

                                                                         

Кнопка «Заполнить»  -  по данной кнопке происходит автоматическое заполнение отчета данными из бухгалтерского учета. При желании пользователь может отредактировать или заполнить данные самостоятельно

 

Кнопка «Очистить» -  очищает бланк отчета

 

Кнопки добавления, удаления строк – пользователь может добавить новую строку (и заполнить ее самостоятельно) или удалить любую выделенную

 

Кнопки последовательного перехода по страницам – пользователь может перейти на следующую / предыдущую страницу отчета; текущая страница и полное количество страниц отображаются между этими кнопками

 

Кнопка «Печать» - позволяет выполнить предварительный просмотр, либо сразу распечатать отчет

 

Кнопка произвольного перехода к странице – пользователь может перейти на любую страницу отчета.

 

Кнопка вызова справки – пользователь может просмотреть текущую справку, либо перейти к общей информации о формах.

  

Кроме этого, в процессе работы с отчетом пользователь может самостоятельно редактировать данные, представленные в бланке отчета. По двойному щелчку на ячейке открывается элемент (карточка контрагента), после редактирования которого обновляются все связанные поля (Резидент\нерезидент,РНН, ИИН, код страны резидентства). При двойном щелчке на ячейке с обычным текстом или числом (например, в поле "Сумма"), открывается окно для прямого редактирования значения.
