План счетов. Бюджетирование

Для хранения данных о планируемой структуре средств и источников финансирования используется специализированный план счетов, открытый для редактирования пользователем. Это «План счетов для бюджетирования». Состав счетов может произвольно изменяться в интересах предприятия (за исключением счета «Служебный» 000).

Необходимость ведения плана счетов бюджетирования (управленческого плана счетов) определяется в параметрах учета.

Названия счетов и суть хранимой на них информации полностью определяется политикой бюджетирования на предприятии. Для счета указывается признак ведения суммовых и количественных оборотов, признак учета валютных сумм, настраивается аналитика. На выбор пользователя предлагается пять заранее предопределенных видов аналитических разрезов:

·            Финансовые показатели;

·            проекты;

·            ЦФО (управленческие подразделения);

·            контрагенты;

·            номенклатура.

Эти разрезы аналитики фактически соответствуют основным аналитическим разрезам бюджетирования. Вручную могут быть созданы дополнительные элементы, подключенные в качестве дополнительных разрезов бюджетирования. Как правило, проводки формируются автоматически при формировании оборотов по финансовым показателям и, соответственно, значения аналитики оборотов будут использованы в качестве значений субконто в проводках. Сценарий планирования, с точки зрения использования механизма плана счетов, играет роль разделителя учета. Таким образом, для каждой проводки, помимо указанных видов аналитики, фиксируется значение сценария планирования.

Список таких разрезов задается в списке «Виды субконто». Здесь же настраивается необходимость хранения остатков по данным разрезам. Количество разрезов, одновременно применяемых к счету, ограничено пятью.

Дополнительно для каждого счета можно указать источник получения данных о количественных и суммовых остатках из учетной системы. Это соответствие может быть в дальнейшем использовано для коррекции остатков на плане счетов бюджетирования по данным учетной системы, а также для формирования входящего баланса при начале планирования. Настройка такого источника происходит по команде «Отложенный сбор факта» верхней командной панели. Эта команда доступна только если в параметрах учета не отключено использование отложенного режима сбора фактических данных.
